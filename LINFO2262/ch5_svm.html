Interactif SVM à Marge Dure — Visualisation

🧒 Tracer la rue la plus large

Imagine deux quartiers séparés par une rue. Tu veux tracer cette rue de façon à laisser le maximum d'espace des deux côtés — pour qu'une maison mal placée ne soit pas dans le mauvais quartier. C'est exactement ce que fait un SVM (Support Vector Machine) !

Le SVM ne cherche pas n'importe quel hyperplan de séparation — il cherche celui qui maximise la marge, c'est-à-dire la distance entre l'hyperplan et les points les plus proches de chaque classe. Ces points les plus proches s'appellent les vecteurs de support.

🎯 Pourquoi maximiser la marge améliore la généralisation ?Suppose que ta frontière passe très près de certains points. Si une nouvelle donnée arrive légèrement différente, elle pourrait se retrouver du mauvais côté → mauvaise classification !

En maximisant la marge, on crée une zone tampon des deux côtés de la frontière. Les nouvelles données doivent être vraiment de l'autre côté pour être mal classifiées. Plus la marge est large, plus le modèle est robuste.

📐 Mathématiquement : Marge = 2/‖w‖. Maximiser la marge = minimiser ‖w‖² (plus facile mathématiquement car convexe !).
🏘️Vecteurs de supportLes points les plus proches de la frontière — ils 'soutiennent' la marge.
📏Marge = 2/‖w‖Plus ‖w‖ est petit, plus la marge est large.
---HyperplanLa frontière de décision : w·x + b = 0
🛡️Zone tamponLa marge protège contre le bruit et les nouvelles données.

  • Le SVM trouve automatiquement la meilleure frontière et les vecteurs de support.
  • Observe : seuls les points sur la marge (les vecteurs de support) comptent ! Les autres peuvent être supprimés sans changer la frontière.
Le SVM est l'un des algorithmes théoriquement les plus solides : il maximise la marge, ce qui est directement lié à la capacité de généralisation. C'est la solution 'optimale' pour la classification linéaire !

Le SVM cherche l'hyperplan qui maximise la marge entre les classes. Les Support Vectors sont les points sur la marge — eux seuls déterminent la frontière.

Interactif Kernel Trick — Données non-linéairement séparables

🧒 Plier l'espace pour que tout devienne linéaire

Imagine que tu as des points rouges au milieu et des points bleus autour (un cercle). Il est impossible de les séparer avec une droite en 2D. Mais si tu ajoutes une troisième dimension (par exemple, la distance au centre), tu peux les séparer avec un plan horizontal ! C'est l'idée du kernel trick.

Le noyau (kernel) permet au SVM de travailler dans un espace de dimension très élevée (parfois infinie !) sans jamais calculer explicitement les coordonnées dans cet espace. On calcule juste les produits scalaires via la fonction noyau — c'est l'astuce !

🎯 Visualise l'astuce avec une feuille de papier📄 Dessine des points sur une feuille (2D). Certains forment un cercle autour d'autres — non séparables linéairement.

✋ Maintenant froisse la feuille (= transformation φ vers un espace plus grand). Les points qui étaient 'dans le cercle' se retrouvent à un niveau différent.

✂️ Tu peux maintenant couper avec un plan plat (= hyperplan linéaire dans le nouvel espace).

🪄 L'astuce : tu n'as pas besoin de savoir comment la feuille est froissée (= calculer φ(x)). Il suffit de calculer K(x,x') = ⟨φ(x), φ(x')⟩ directement !
🔵🔴Non séparable (2D)Un cercle ne peut pas être séparé par une droite.
φ(x)TransformationOn 'monte' dans un espace de dimension supérieure.

✂️Séparable (nD)Dans le nouvel espace, une ligne droite suffit !
Kernel RBF : K(x,x') = exp(-‖x-x'‖²/2σ²). Si σ est petit → frontière très locale (overfitting possible). Si σ est grand → frontière lisse. Pareil pour C ! Toujours choisir via cross-validation.

Quand les données ne sont pas linéairement séparables, on les projette dans un espace de plus grande dimension via une fonction φ. Le kernel K(x,x') = φ(x)^T φ(x') calcule ce produit scalaire sans calculer φ explicitement !

Remplacer x^T x par K(x, x') dans le dual
Linéaire : K = x^T x' | Polynomial : K = (x^T x' + 1)^d | RBF : K = exp(-||x-x'||²/(2σ²))

Interactif Paramètre C — Marge Souple

🧒 Combien d'erreurs acceptes-tu ?

Dans le vrai monde, les données sont rarement parfaitement séparables. Il y a du bruit, des outliers, des exceptions. Le SVM à marge souple (soft margin) introduit un paramètre C qui contrôle le compromis entre une marge large et peu d'erreurs.

Grand C : le modèle penalise beaucoup les erreurs → essaie de bien classer tous les points → marge plus étroite → risque d'overfitting.
Petit C : le modèle tolère quelques erreurs → marge plus large → meilleure généralisation sur de nouvelles données.

🎯 L'analogie du prof strict vs cool👨‍🏫 Prof strict (C grand) : 'Tu dois avoir tout juste, 0 erreur tolérée.' → Les étudiants mémorisent tout par cœur mais échouent aux nouvelles questions (overfitting).

😎 Prof cool (C petit) : 'Quelques erreurs ok, l'important c'est de comprendre.' → Les étudiants comprennent les principes et réussissent les nouvelles questions (bonne généralisation).

🏆 Le bon C dépend de tes données. On le choisit par cross-validation : on teste plusieurs valeurs de C et on garde celle qui donne le meilleur score sur les données de validation.
  • Ajuste le slider C : observe comment la marge et les violations changent.
  • C très grand (1000) : presque hard-margin, sensible aux outliers.
  • C très petit (0.01) : marge large, beaucoup d'erreurs tolérées.
  • C optimal : trouver le sweet spot via cross-validation sur tes vraies données.
Le SVM a deux hyperparamètres principaux à tuner : C (marge souple) et σ pour le noyau RBF. Une grille de recherche (grid search) avec cross-validation est la méthode standard.

Le paramètre C contrôle le compromis biais-variance dans le SVM à marge souple. C grand = marge plus stricte (risque overfitting). C petit = plus de violations tolérées (risque underfitting).
