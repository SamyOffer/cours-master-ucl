⚠️ Le grand défi

Le problème du sous-graphe isomorphe est NP-complet ! C'est beaucoup plus difficile que la correspondance d'itemsets (qui est polynomial).

Cela signifie qu'il n'existe pas d'algorithme connu pour le résoudre rapidement en général. Nous devons utiliser des heuristiques et des prunings.

Section 3 : Approche Apriori pour les Graphes (AGM/FSG)

Apriori sur les graphes

Idée clé : L'idée d'Apriori s'applique aussi aux graphes : si un sous-graphe est infrequent, tout sur-graphe qui le contient est aussi infrequent !

Propriété Apriori pour les graphes

Deux approches classiques

1. AGM (Adjacency Matrix)

Génère les candidats en fusionnant les graphes sommet par sommet. Plus simple à implémenter mais moins efficace.

2. FSG (Frequent Subgraph)

Génère les candidats en ajoutant des arêtes une par une. Meilleure performance en pratique.

Étapes de l'algorithme

  1. Trouver les sous-graphes fréquents à 1 nœud (L₁)
  2. Trouver les sous-graphes fréquents à 1 arête (L₂)
  3. Générer les candidats à k+1 à partir des k-sous-graphes fréquents
  4. Élaguer les candidats infrequents
  5. Répéter jusqu'à ce qu'aucun candidat ne soit fréquent

🔑 Problème clé : forme canonique

Un même graphe peut avoir de nombreuses représentations différentes. Nous avons besoin d'une forme canonique pour éviter de générer le même graphe plusieurs fois.

Exemple : A-B-C peut être écrit (A,B,C) ou (C,B,A) — même graphe, deux écritures !

Section 4 : gSpan — Pattern-Growth pour les Graphes

Pattern-growth

Idée : gSpan est comme PrefixSpan mais pour les graphes : au lieu d'ajouter des items à une séquence, on ajoute des arêtes à un graphe !

Innovations clés de gSpan

1. DFS Code

Une représentation canonique des graphes utilisant un parcours en profondeur (DFS).

2. Rightmost Extension

On n'ajoute des arêtes que sur le chemin le plus à droite de l'arbre DFS. Cela garantit que chaque graphe est généré exactement une seule fois !

3. Ordre lexicographique

Définir un ordre < sur les codes DFS pour éviter les doublons.

DFS Code : liste ordonnée des arêtes dans l'ordre du parcours DFS

Règle de Rightmost Extension

Pour étendre un pattern, nous pouvons seulement ajouter :

  • Une arête vers l'arrière (backward edge) du nœud le plus à droite vers un ancêtre antérieur
  • Une arête vers l'avant (forward edge) d'un nœud sur le chemin le plus à droite

Pourquoi ? Cela garantit que chaque graphe est généré exactement une fois !

Étapes de l'algorithme gSpan

  1. Commencer par toutes les arêtes uniques fréquentes
  2. Pour chaque pattern fréquent p, l'étendre en utilisant la règle rightmost
  3. Vérifier si l'extension est canonique (minimale lexicographiquement)
  4. Si canonique ET fréquente : ajouter aux résultats et récurser

Variantes importantes

Mining dans un seul graphe volumineux

Au lieu d'une base de données de petits graphes, nous avons UN très gros graphe (ex: le Web).

Subgraph mining discriminatif

Trouver les sous-graphes qui distinguent bien les deux classes. Utile pour la classification basée sur les graphes.

Subgraph mining contraint

Imposer des contraintes sur les patterns (ex: "seulement les cycles", "seulement les arbres").

Approximate subgraph mining

Permettre les petites différences (inexact matching) pour être plus robuste au bruit.
