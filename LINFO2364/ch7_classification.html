🎯 Chapitre 7 — Classification

Section 1 : C'est quoi la Classification ?

C'est quoi la classification ?

Analogie simple : Pense à un médecin qui examine des symptômes (fièvre, toux, fatigue) et doit décider si c'est une grippe, un rhume, ou le COVID. La classification = apprendre automatiquement ces règles à partir de milliers de cas !

Les deux phases

🤖

🆕
Nouvelles données
🎯
Prédiction

Phase d'apprentissage vs Phase de classification

1. Phase d'apprentissage (Training)

Le classifier apprend un modèle à partir de données étiquetées : chaque exemple a un label connu.

2. Phase de classification (Testing)

Utiliser le modèle appris pour prédire les labels de nouvelles données (sans labels).

Exemples concrets

💰 Banque
Prêt "risqué" ou "sûr" ?
🍄 Champignon
"Comestible" ou "vénéneux" ?
📧 Email
"Spam" ou "pas spam" ?
👨‍⚕️ Médecin
"Traitement A", "B" ou "C" ?

Classification vs Régression

  • Classification : Prédire une catégorie/label discret (ex: {spam, pas spam})
  • Régression : Prédire une valeur numérique continue (ex: prix d'une maison)

Section 2 : Arbres de Décision

Arbres de décision

Analogie : Un arbre de décision c'est comme un jeu de 20 questions. À chaque nœud, on pose une question sur les données. Selon la réponse, on va à gauche ou à droite jusqu'à atteindre une feuille (la prédiction) !

Exemple : classifier les emails

Est-ce que l'email contient "gratuit" ? ├── OUI → Est-ce qu'il contient "urgent" ? │ ├── OUI → SPAM ✗ │ └── NON → Est-ce qu'il vient d'un contact ? │ ├── OUI → PAS SPAM ✓ │ └── NON → SPAM ✗ └── NON → PAS SPAM ✓

Composants d'un arbre

  • Nœud interne : test sur un attribut (question)
  • Arête : valeur de l'attribut (réponse)
  • Feuille : classe prédite (destination finale)

Avantages des arbres de décision

  • ✅ Très interprétables (on peut lire les règles)
  • ✅ Pas besoin de normaliser les données
  • ✅ Gère bien les données catégorielles
  • ⚠️ Risque de surapprentissage (overfitting)

Section 3 : De l'Arbre aux Règles

Conversion en règles

Idée : Un arbre de décision peut être converti en règles IF-THEN. Chaque chemin de la racine à une feuille = une règle !

Section 4 : Algorithme Sequential Covering

Sequential Covering

Idée : L'algorithme de couverture séquentielle apprend les règles une par une. Pour chaque règle apprise, on retire les exemples qu'elle couvre, et on répète !

Étapes de l'algorithme

  1. Initialiser : liste de règles = [], exemples restants = D
  2. Tant que D n'est pas vide ET critère d'arrêt non atteint :
    • a) Apprendre la meilleure règle r couvrant certains exemples dans D
    • b) Ajouter r à la liste de règles
    • c) Retirer de D les exemples couverts par r
  3. Retourner la liste de règles

Comment apprendre "la meilleure règle"

Stratégie : Beam Search

  • Commencer avec une règle vide (couvre tout)
  • Ajouter itérativement des conditions pour maximiser l'accuracy
  • Arrêter quand la règle couvre assez d'exemples avec une bonne accuracy

Section 5 : Classification Associative (CBA)

Règles d'association pour la classification

Idée brillante : Utiliser les règles d'association (comme dans le chapitre 2) pour faire de la classification ! On mine des règles comme {symptôme1, symptôme2} ⇒ {maladie} directement !

Étapes de CBA

  1. Miner les règles d'association : Condition ⇒ Class (avec support et confidence min)
  2. Trier les règles par priorité (confidence, puis support, puis longueur)
  3. Pour une nouvelle exemple, appliquer la première règle qui correspond

Exemple : prédire les maladies

Règle 1 :
{fièvre > 38, toux} ⇒ grippe
Confidence: 92% | Support: 15%
Règle 2 :
{fièvre > 38} ⇒ infection
Confidence: 75% | Support: 30%
Règle 3 (défaut) :
{} ⇒ bien portant
Confidence: 60%

Avantages de CBA

  • ✅ Très interprétable (les règles sont lisibles)
  • ✅ Utilise les algorithmes efficients de mining (Ch2-4)
  • ✅ Bonne accuracy en pratique
  • ✅ Peu de surapprentissage

Section 6 : Évaluation d'un Classifieur

Métriques principales

Accuracy (Précision globale)

Precision (Précision)

Parmi les exemples prédits positifs, combien sont réellement positifs ?

Recall (Sensibilité/Rappel)

Parmi les exemples vraiment positifs, combien avons-nous correctement identifiés ?

F1-Score (Moyenne harmonique)

Balance entre Precision et Recall.

Légende : TP, TN, FP, FN

  • TP (True Positive) : Correctement prédit comme positif
  • TN (True Negative) : Correctement prédit comme négatif
  • FP (False Positive) : Incorrectement prédit comme positif
  • FN (False Negative) : Incorrectement prédit comme négatif
