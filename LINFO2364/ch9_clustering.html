Chapitre 9 : Clustering (Regroupement)

Chapitre 9
Qu'est-ce que le Clustering ? Le clustering regroupe des objets similaires ensemble sans étiquettes prédéfinies. Contrairement à la classification (Ch7), nous ne connaissons pas les classes à l'avance. C'est du learning non supervisé.

Exemples d'applications :

📊 Segmentation de clients en groupes de profils similaires
🔬 Classification de gènes similaires
🗺️ Découverte de régions géographiques denses
💬 Regroupement de documents similaires

Défi principal : Comment mesurer la "similarité" ? Comment choisir le nombre de clusters ?

Section 1 : K-means et Variantes

K-means : Partitionnement Simple et Efficace

Principe : Partitionner n objets en k clusters disjoints (k ≤ n) pour minimiser la distance moyenne de chaque point à son centroïde.

Formule d'optimisation

Algorithme K-means

  1. Initialisation : Sélectionner aléatoirement k objets comme centroïdes initiaux
  2. Assignation : Assigner chaque point au centroïde le plus proche
  3. Mise à jour : Recalculer les centroïdes comme la moyenne de chaque cluster
  4. Répétition : Répéter les étapes 2-3 jusqu'à convergence (centroïdes stables)

Exemple numérique concret

Points : P = {(1,1), (2,2), (5,5), (6,6)}

k=2

Itération 1 : Centroïdes initiaux = (1,1) et (5,5)

Cluster 1 : {(1,1), (2,2)} → nouveau centroïde = (1.5, 1.5)

Cluster 2 : {(5,5), (6,6)} → nouveau centroïde = (5.5, 5.5)

Itération 2 : Les centroïdes ne bougent plus → Convergence!

Caractéristiques

Aspect Description
Complexité O(nkt) ≈ linéaire (n=objets, k=clusters, t=itérations)
Avantages Simple, scalable, convergence locale garantie, efficace
Limites k manuel, dépend de l'initialisation, sensible aux outliers, mauvais en haute dimension

Section 2 : Estimer le Nombre de Clusters

Index de Calinski-Harabasz

Problème : Comment choisir le meilleur k ? L'index CH mesure la qualité d'un clustering : haute séparation entre clusters + haute compacité intra-cluster.

Formule CH(k)

Variance entre clusters (séparation)

Variance intra-cluster (compacité)

Maximiser CH(k) pour trouver le meilleur k !

Interprétation

  • CH élevé = clusters bien séparés et compacts
  • CH faible = clusters mal séparés ou dispersés
  • Trouver le k où CH atteint son maximum

Section 3 : Clustering Hiérarchique

Deux Approches

Agglomératif (Bottom-Up) : Commence par n clusters (1 objet chacun) et fusionne itérativement les clusters les plus proches.

Divisif (Top-Down) : Commence par 1 cluster (tous les objets) et le divise récursivement.

Mesures de Distance Entre Clusters

Méthode Formule Caractéristique
Single Linkage dist(Ci, Cj) = min distance Peut créer des chaînes longues
Complete Linkage dist(Ci, Cj) = max distance Clusters compacts et sphériques
Average Linkage dist(Ci, Cj) = moyenne distance Bon compromis, robuste
Ward's Method Minimise variance intra-cluster Similaire à K-means mais hiérarchique

Résultat : Le Dendrogramme

Un dendrogramme est un arbre qui montre comment les objets sont progressivement fusionnés ou divisés, avec les distances de fusion en ordonnée.

Section 4 : Clustering Density-Based (DBSCAN)

Principe Fondamental

DBSCAN : Regroupe les points en fonction de leur densité locale, pas en fonction d'une distance globale. Découvre des clusters de forme arbitraire et identifie les outliers.

Paramètres

  • ε (epsilon) : Rayon de voisinage autour d'un point
  • MinPts : Nombre minimum de points dans le rayon ε pour qu'un point soit un core object

Définitions Clés

  • Core object : Un point avec ≥ MinPts points dans son rayon ε (y compris lui-même)
  • Border point : Un point non-core mais dans le rayon d'un core object
  • Noise (Outlier) : Un point qui n'est ni core ni border
  • Cluster : Ensemble connecté de core objects + border points

Algorithme DBSCAN

  1. Marquer tous les points comme non-visités
  2. Sélectionner un point non-visité aléatoirement
  3. Si c'est un core object → créer un nouveau cluster et l'étendre avec tous les voisins accessibles
  4. Si c'est un border/noise → passer au suivant
  5. Répéter jusqu'à tous les points visités

Avantages vs Limites

Avantages Limites
✅ Ne nécessite pas de k ⚠️ Sensible au choix de ε et MinPts
✅ Découvre clusters arbitraires (non-sphériques) ⚠️ Mauvais avec densités variables
✅ Détecte les outliers ⚠️ O(n²) dans le pire cas

HDBSCAN : Variante Hiérarchique

HDBSCAN : Amélioration de DBSCAN qui ne nécessite qu'un paramètre (MinPts) au lieu de deux, et gère les densités variables.

Concepts clés

  • Core distance dcore(p) : Distance au MinPts-ème voisin
  • Mutual reachability distance : dmreach(p,q) = max{dcore(p), dcore(q), d(p,q)}
  • Construit un Minimum Spanning Tree, puis une hiérarchie de clusters

Section 5 : Évaluation du Clustering

Hopkins Statistic : Tester la Clusterabilité

Question : Les données contiennent-elles vraiment des clusters, ou sont-elles uniformément distribuées aléatoirement ?

Formule Hopkins Statistic

Où xi = distance du point uniforme aléatoire au plus proche point réel, yi = distance du point réel au plus proche point réel.

Interprétation

  • H ≈ 0.5 : Données uniformément distribuées (pas de clusters)
  • H → 1.0 : Données fortement clusterisées (clusters clairs)
  • H ≈ 0 : Cas dégénéré (clusters très denses)

Dunn Index : Séparation vs Compacité

Idée : Maximiser la distance entre clusters (séparation) tout en minimisant les distances intra-cluster (compacité).

Formule

δ = distance minimale inter-clusters (séparation)

Δ = distance maximale intra-cluster (diamètre)

Plus grand DI = meilleur clustering

Silhouette Coefficient : Mesure Complète

Pour chaque point : Comparer sa distance à son propre cluster vs sa distance aux autres clusters.

Formules

Interprétation du Silhouette

  • s(o) ≈ 1 : Point bien classé dans son cluster
  • s(o) ≈ 0 : Point à la limite entre deux clusters
  • s(o) ≈ -1 : Point probablement mal classé

Section 6 : Fuzzy Clustering

Clustering avec Appartenance Floue

Au lieu de : Chaque point appartient à exactement un cluster.
Fuzzy Clustering : Chaque point a un degré d'appartenance (entre 0 et 1) pour chaque cluster !

Formulation

Objectif à Minimiser

p ≥ 1 : paramètre de "flou" (fuzziness)

p=1 : dur (0 ou 1), p→∞ : très fuzzy

Algorithme : Expectation-Maximization (EM)

  1. E-step : Assigner les poids wij en fonction des distances actuelles
  2. M-step : Minimiser SSE en mettant à jour les centroïdes
  3. Répéter jusqu'à convergence

Formule de Mise à Jour des Poids (k=2 clusters)

Plus un point est proche du cluster 1, plus wi1 est proche de 1.

Exemple

Point p entre deux clusters C1 et C2 :

dist(p, C1) = 3, dist(p, C2) = 5

wp,C1 = 25/(9+25) = 0.735 (73.5% dans C1)

wp,C2 = 9/(9+25) = 0.265 (26.5% dans C2)

Avantages et Cas d'Usage

  • ✅ Plus flexible : représente l'incertitude
  • ✅ Bon pour les clusters qui se chevauchent
  • ⚠️ Plus complexe à interpréter
  • ⚠️ Applications : segmentation d'images, classification d'ADN

Section 7 : Clustering Avancé

Clustering de Données Haute Dimension

Problème : En haute dimension (centaines de features), la plupart des points sont loin les uns des autres (malédiction de la dimensionnalité).

Solutions

  • Réduction de dimension : PCA, t-SNE, UMAP avant clustering
  • Subspace clustering : Cluster dans des sous-espaces de features
  • Weighted features : Donner moins de poids aux features bruitées

Biclustering : Clustering sur 2 Dimensions

Problème : On veut grouper à la fois des lignes (objets) et des colonnes (features) simultanément.

Exemple : Matrice Clients × Produits

Film A Film B Film C
Client 1 5 4 1
Client 2 5 3 1
Client 3 1 1 5

Bicluster : {Client 1,2} × {Film A,B} : clients aiment les films A,B

Types de Biclusters

  • Constant values : eij = c
  • Constant rows : eij = c + αi (même profil par ligne)
  • Constant columns : eij = c + βj (même profil par colonne)
  • Coherent values : eij = c + αi + βj (combinaison additif)

δ-bicluster

Mesure la variance résiduelle. δ faible = bon bicluster cohérent.

Clustering de Graphes et Réseaux

Problème : Grouper les nœuds d'un graphe pour détecter les communautés.

Métrique : Modularité

Mesure l'excès de liens dans les clusters vs ce qu'on attendrait aléatoirement.

Exemples

  • 🔗 Détection de communautés dans les réseaux sociaux
  • 🧬 Groupes de gènes qui interagissent
  • 📱 Détection de fraude bancaire (motifs d'interactions suspectes)

Semi-Supervised Clustering

Idée : Utiliser quelques labels partiels pour guider le clustering.

Approche

  • Certains points ont des labels (partiellement supervisé)
  • Autres points sont non-labelisés
  • L'algorithme doit regrouper les points non-labelisés en respectant les contraintes des points labelisés

Clustering de Patterns Compressés/Approximatifs

Objectif : Découvrir des patterns qui se répètent approximativement (avec variations).

Exemple : Time-Series Clustering

Grouper les séries temporelles avec comportements similaires (même si pas identiques).

Section 8 : Autres Méthodes de Clustering

CLIQUE : Clustering Grid-Based

Principe : Diviser l'espace en une grille de cellules. Les cellules denses forment les clusters.

Avantages

  • ✅ Efficace (complexité linéaire en général)
  • ✅ Clusters de forme arbitraire
  • ✅ Découvre aussi les clusters dans les sous-espaces

Limites

  • ⚠️ Sensible à la taille de la grille
  • ⚠️ Densités différentes par dimension

Comparaison des Méthodes

Méthode Type Paramètres Formes découvertes Outliers
K-means Partitionnement k Sphériques Non
DBSCAN Density-based ε, MinPts Arbitraires Oui
Hiérarchique Agglomératif Aucun (dendrogramme) Arbitraires Non
Fuzzy C-means Partitionnement k, p (fuzziness) Sphériques Non
CLIQUE Grid-based Taille grille Arbitraires Oui

Bonnes Pratiques

  1. Prétraiter les données : Normaliser, enlever outliers extrêmes
  2. Tester plusieurs méthodes : K-means, DBSCAN, hiérarchique
  3. Évaluer la qualité : Hopkins, Silhouette, visibilité
  4. Valider avec experts : Les clusters ont-ils du sens métier ?
  5. Considérer la scalabilité : Gros dataset → K-means, CLIQUE
