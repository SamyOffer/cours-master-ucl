🔍 Chapitre 10 — Détection d'Anomalies (Outlier Detection)

Introduction : C'est quoi une anomalie ?

Un problème universel

Exemple concret : Une banque monitore les transactions d'un client. Habituellement, vous dépensez 50-100€ par jour, localement, en euros. Un jour, vous faites un achat de 5000€ à Singapour en devises étrangères. C'est suspect ! 🚨 C'est une anomalie car ce comportement dévie significativement de vos patterns habituels.

Applications importantes

💰 Détection de fraude
Transactions suspectes, accès frauduleux
🔒 Sécurité publique
Comportements suspects, intrusions réseau
⚙️ Pannes industrielles
Machines qui dysfonctionnent, capteurs défaillants
📹 Vidéosurveillance
Actions anormales, événements suspects

Lien avec le Clustering

Vous vous rappelez du clustering (chapitres 8-9) ? Il trouve les patterns majoritaires (les clusters). La détection d'anomalies fait l'inverse : elle capture les cas exceptionnels qui ne correspondent à aucun pattern.

  • Clustering : "Voici les 3 groupes de clients habituels"
  • Outlier Detection : "Voici les 5 clients anormaux qui ne rentrent dans aucun groupe"

Outlier ≠ Bruit

Attention distinction importante !

  • Bruit : Erreur aléatoire (capteur défaillant, typo dans les données). C'est du charabia, pas intéressant.
  • Outlier : Donnée rare mais significative. C'est générée par un mécanisme différent et c'est exactement ce qu'on veut trouver !

Exemple : Acheter pour 200€ au lieu de 100€ au supermarché habituel = probablement un achat exceptionnel (anniversaire, etc.). Acheter 50000€ dans des magasins jamais visités avec carte clonée = fraude avérée. Le deuxième est un outlier significatif !

Section 1 : Concepts Fondamentaux

Définition formelle

Outlier (Anomalie) : Un objet qui dévie significativement des autres objets, comme s'il était généré par un mécanisme stochastique différent.

C'est la clé : les outliers ne viennent pas du même "processus générateur" que les données normales.

Importance de la justification

Simplement dire "c'est une anomalie car c'est loin des autres" n'est pas suffisant. Il faut justifier POURQUOI cet objet vient d'un processus différent. Sinon, c'est peut-être juste du bruit ou une donnée rare mais normale.

Bon : "Cette transaction est anormale car elle vient d'un pays étranger, avec un type d'achat différent ET un montant 100x plus grand" (plusieurs indices)

Mauvais : "Cette transaction est anormale car elle est un peu plus grande que d'habitude" (pas assez de preuves)

Catégories d'outliers

🌍
Outliers globaux
🎯
Outliers locaux
👥
Outliers collectifs
  • Outlier global : Très éloigné de tous les autres (facile à détecter)
  • Outlier local : Normal dans sa région, anormal comparé au reste (plus difficile)
  • Outlier collectif : Séquence d'objets normaux individuellement mais anormale collectivement

Section 2 : Approches Statistiques (Paramétriques)

Hypothèse : normalité

Les méthodes statistiques supposent que les données normales sont générées par un processus probabiliste (ex: distribution gaussienne). Les objets dans les régions de faible probabilité sont des outliers.

Méthode 1D : Règle 3-sigma

Idée simple : Si la donnée suit une loi normale, 99.7% des données se trouvent dans [μ - 3σ, μ + 3σ].

Section 3 : Approches Statistiques (Non-paramétriques)

Pas d'hypothèse a priori

On ne suppose pas que les données suivent une distribution normale. On laisse les données décider du modèle !

Méthode par Histogramme

Idée : Diviser l'espace en "bins" (intervalles). Un point dans un bin vide ou très peu dense = outlier.

Algorithme :

  1. Construction : Créer un histogramme avec les données d'entraînement
  2. Test : Pour chaque donnée de test, vérifier si elle tombe dans un bin
  3. Score : Les points en dehors des bins sont des outliers

Score d'outlier :

Score élevé = très peu dense = probable outlier !

Problème critique : Taille des bins

  • Bins trop petits : Beaucoup de faux positifs (donne trop d'outliers)
  • Bins trop grands : Pas assez de sensibilité (manque d'outliers)

Section 4 : Détection basée sur la Distance

Idée simple : isolement

Un objet entouré de voisins proches = normal. Un objet isolé = outlier. On regarde si un objet a assez de voisins dans un rayon r.

Définition formelle : DB(r, π)-outlier

Un objet o est un DB(r, π)-outlier si :

En français : si le nombre de voisins à distance r est ≤ π% de la population → outlier !

Paramètres : r = rayon, π = pourcentage seuil

Avantages et limitations

  • ✅ Bon pour détecter les outliers globaux
  • ❌ Problème avec densités variables : un point peut être normal dans une région dense et outlier dans une région sparse

Section 5 : Local Outlier Factor (LOF) - Approche par Densité

Amélioration critique

La distance-based detection échoue avec des densités variables. LOF résout ça en comparant la densité locale d'un point avec ses voisins !

Les concepts clés

1. k-distance : Distance au k-ème plus proche voisin

2. Voisinage k :

Tous les points à distance ≤ dist_k(o) de o

3. Reachability Distance : Distance "lissée" qui tient compte du voisinage du point cible

4. Local Reachability Density : Moyenne des reachability distances

Densité locale = nombre de voisins / distance moyenne

5. Local Outlier Factor :

Rapport entre densité locale du point et densité moyenne de ses voisins

Interprétation du LOF

  • LOF ≈ 1 : Densité similaire aux voisins → point normal
  • LOF < 1 : Plus dense que les voisins → point central du cluster (normal)
  • LOF > 1 : Moins dense que les voisins → point isolé → outlier potentiel ! 🚨

Heuristique : Si LOF > 1.5, c'est probablement un outlier

Section 6 : Approches par Apprentissage

Approche 1 : Clustering-Based (Unsupervised)

Utiliser les clusters trouvés par k-means ou autres pour identifier les outliers.

3 stratégies :

  1. Non-apparteneur : L'objet n'appartient à aucun cluster → outlier
  2. Distance au cluster : Objet très éloigné du cluster le plus proche → outlier
  3. Petit cluster : Cluster très petit ou sparse → tous ses points sont des outliers potentiels

Ratio de distance :

Score élevé = éloigné du cluster = outlier probable

FindCBLOF (Cluster-Based Local Outlier Factor) :

  1. Trouver les clusters avec k-means
  2. Trier par taille décroissante, définir seuil α
  3. Pour chaque point : si dans grand cluster, score = taille × similarité. Si dans petit cluster, score = taille × similarité au cluster le plus proche
  4. Petits scores → outliers

Approche 2 : Classification-Based (Supervised)

Cas important : on a des exemples de la classe "normal" mais rarement des exemples d'outliers. Solution : One-Class Classification

One-Class SVM / Isolation Forest / Autoencoder : Modéliser uniquement la classe "normale". Ce qui dévie du modèle = outlier !

Approche 3 : Semi-Supervised

Mélange : on a quelques exemples d'outliers ET beaucoup d'exemples normaux. Utiliser ces deux informations pour construire un meilleur modèle.

Section 8 : Détection en Haute Dimension

La malédiction de la dimensionnalité

Problème majeur : Quand on a 100, 1000, 10000 dimensions (variables), les distances deviennent toutes similaires ! Tous les points semblez être à peu près à la même distance de tous les autres points.

Pourquoi c'est un problème ?

  • Distance-based : Les distances deviennent inutiles (tous les points sont "à peu près aussi loin")
  • LOF : K-ème plus proche voisin n'a plus de sens
  • Histogramme : Volume des bins explose exponentiellement

Solutions

  1. Réduction de dimension (PCA) : Projet dans un sous-espace de dimension faible, puis détecte les outliers dans ce sous-espace
  2. Approches subspace : Chercher les sous-espaces où l'anomalie est évidente
  3. Méthodes spécialisées : Isolation Forest, LOF multi-local, etc.
