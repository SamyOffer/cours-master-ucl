Section 1 : Bayesian Belief Networks (BBN)

Qu'est-ce qu'un BBN ?

Idée clé : Un BBN est un modèle graphique probabiliste qui représente comment les variables dépendent les unes des autres. Par exemple : "Faut-il utiliser un parapluie ?" dépend de "La météo est-elle nuageuse ?" qui dépend de "Est-ce humide le matin ?"

Structure et concepts fondamentaux

Graphe acyclique dirigé (DAG)

Un BBN est représenté par un DAG où :

  • Nœuds : variables aléatoires
  • Arêtes : dépendances causales conditionnelles
  • Acyclique : pas de cycles (pas de bouclages)

Probabilités conditionnelles

Chaque nœud a une table de probabilités conditionnelles (CPT) : P(nœud | parents du nœud)

Factorisation de la probabilité

Exemple concret : Maladie et Symptômes

Variables :

  • A = "Patient a une grippe" : P(A=1) = 0.4
  • B = "Patient a une allergie" : P(B=1) = 0.2
  • C = "Patient a une toux" : dépend de A et B

Table de probabilités conditionnelles P(C|A,B) :

A B P(C=1|A,B) P(C=0|A,B)
0 0 0.05 0.95
0 1 0.25 0.75
1 0 0.80 0.20
1 1 0.95 0.05

Section 2 : Évaluation des Modèles et Métriques

Comment mesurer la qualité d'un modèle ?

Problème : L'accuracy n'est pas suffisante ! Si 99% des emails ne sont pas spam, dire "aucun email n'est spam" donne 99% d'accuracy mais zéro utilité. Il faut d'autres métriques.

Les 4 métriques principales

1. Accuracy (Précision globale)

Interprétation : Proportion de prédictions correctes parmi tous les cas.

Attention : Trompeuse si les classes sont déséquilibrées.

2. Precision (Précision positive)

Interprétation : Parmi les cas prédits positifs, combien sont vraiment positifs ?

Utilité : Importante quand les faux positifs sont chers (ex: alerte cancer).

3. Recall (Sensibilité/Rappel)

Interprétation : Parmi les vrais cas positifs, combien avons-nous détectés ?

Utilité : Importante quand les faux négatifs sont chers (ex: détection fraude).

4. F1-Score (Moyenne harmonique)

Interprétation : Balance entre Precision et Recall (0 à 1, plus haut = mieux).

Utilité : Métrique unique pour résumer la qualité quand precision et recall importent tous deux.

Section 3 : Division du Dataset (Data Splitting)

Pourquoi diviser les données ?

Problème du surapprentissage : Si on entraîne ET teste sur les mêmes données, le modèle peut mémoriser les réponses au lieu d'apprendre les règles. Il faut données d'entraînement et données de test séparées.

Approche 1 : Holdout (Simple Split)

Idée : Diviser aléatoirement en deux : 70% entraînement, 30% test.

  • Avantage : Simple et rapide
  • Inconvénient : Résultat dépend du hasard de la division. Si on a peu de données, peut être instable.

Approche 2 : k-Fold Cross Validation

Idée : Diviser en k parties égales, entraîner k fois avec k-1 parties et tester sur 1 partie.

  • Exemple avec k=5 : 5 expériences, chacune utilisant 80% pour train et 20% pour test
  • Résultat final : Moyenne des k scores
  • Avantage : Robuste, utilise toutes les données
  • Cas extrême (LOO) : k = n (Leave-One-Out) : chaque point est testé seul

Section 4 : Feature Selection

Pourquoi sélectionner les features ?

Problème : Avec 10,000 features, on ne peut pas tout garder (bruit, surapprentissage, lenteur). Il faut enlever les features inutiles.

Les 3 approches principales

1. Filter Methods (Avant la classification)

Idée : Évaluer chaque feature indépendamment (ex: Chi-square, corrélation) puis garder les k meilleures.

  • Avantage : Très rapide
  • Inconvénient : Ignore les interactions entre features

2. Wrapper Methods (Utilisant le classifieur)

Idée : Chercher le meilleur sous-ensemble de features en entraînant le classifieur plusieurs fois.

  • Forward Selection : Commencer sans features, en ajouter une par une (celle qui améliore le plus)
  • Backward Selection : Commencer avec toutes, enlever une par une (celle qui nuit le moins)
  • Avantage : Utilise le vrai classifieur
  • Inconvénient : Très coûteux en temps

3. Embedded Methods (Pendant l'apprentissage)

Idée : L'algorithme d'apprentissage sélectionne les features naturellement.

  • Exemple : Arbres de décision (Ch6) montrent quelles features sont les plus importantes
  • Avantage : Très rapide, sélection intelligente

Section 5 : Classification avec Weak Supervision

Quand on n'a pas assez de données labellisées ?

Réalité : Annoter manuellement 1 million d'images coûte cher et prend du temps. On veut utiliser les données non-labellisées aussi.

Approche 1 : Semi-Supervised Learning

Idée : Utiliser quelques données labellisées (L) + beaucoup non-labellisées (U).

  • Entraîner un premier modèle sur L
  • Prédire les labels de U (avec confiance)
  • Ajouter les prédictions confiantes à L
  • Réentraîner et répéter (Self-Training)

Approche 2 : Active Learning

Idée : Demander intelligemment à un expert de labelliser les cas les plus informatifs.

  • Stratégie d'incertitude : Demander les cas où le modèle est moins sûr
  • Exemple : Un modèle de diagnostic médical demande "Expert, qu'est-ce que cette image ?" pour les cas ambigus
  • Avantage : Moins d'annotations mais plus informatives

Tableau comparatif : Supervision

Type Données labellisées Données non-labellisées Utilité
Supervised Beaucoup Non Performance optimale, mais coûteux
Semi-Supervised Peu Beaucoup Bon trade-off coût/performance
Active Learning Peu (choisis intelligemment) Beaucoup Minimise le nombre d'annotations
Unsupervised Aucun Beaucoup Clustering, pas de classification

Section 6 : Classification avec Rich Data Types

Au-delà des nombres : texte, images, séries temporelles

Défi : La plupart des données ne sont pas des nombres ! Comment classer du texte ou des images ?

1. Text Classification

Approche : Bag of Words + TF-IDF

  • Étape 1 : Construire un dictionnaire de mots (ex: 10,000 mots uniques)
  • Étape 2 : Pour chaque document, compter la fréquence des mots
  • Étape 3 : Normaliser par TF-IDF (Term Frequency - Inverse Document Frequency)
  • Résultat : Un vecteur numérique par document → classification standard

Exemple :

Document Fréquence "bon" Fréquence "mauvais" TF-IDF "bon" TF-IDF "mauvais" Label
Review 1 5 0 0.78 0.00 Positif
Review 2 1 4 0.15 0.62 Négatif

2. Image Classification

Approche moderne : Deep Learning (Réseaux de neurones convolutifs)

  • L'image est un tensor 3D (hauteur × largeur × couleurs)
  • Les couches de convolution extraient des features visuelles (edges, textures, objets)
  • Les dernières couches font la classification
  • Entraîné sur millions d'images labellisées

Résultats : Très bonne accuracy, souvent meilleur que humain !

3. Time Series Classification

Données temporelles : Séquences de valeurs dans le temps (ECG, stock price, température).

  • Défi : L'ordre temporel important (pas comme bag of words)
  • Approches : DTW (Dynamic Time Warping), LSTM (Long Short-Term Memory), etc.
  • Exemple : Classer un ECG comme "normal" ou "arythmie" en regardant la forme de la courbe

Section 7 : Sélection de Modèles (Model Selection)

Quel modèle est vraiment le meilleur ?

Question : Si Modèle A a accuracy 82% et Modèle B a 83.5%, la différence est-elle significative ou due au hasard ?

Solution : t-test statistique

Idée : Comparer deux modèles avec un k-fold cross-validation et utiliser un test t.

Étapes :

  1. Faire un 5-fold cross-validation pour chaque modèle
  2. On obtient 5 scores (un par fold) pour le modèle A, et 5 pour le modèle B
  3. Calculer la différence moyenne : δ = mean(scores_A) - mean(scores_B)
  4. Calculer la statistique t avec k-1 degrés de liberté (ici: 4)
  5. Comparer à la valeur critique (ex: σ = 5%, t_critique ≈ 2.776)
  6. Si |t_calculé| > t_critique, les modèles sont statistiquement différents
