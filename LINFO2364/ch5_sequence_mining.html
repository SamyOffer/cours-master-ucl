Section 1: C'est quoi une Séquence ?

Définition

Analogie historique: Imagine l'historique d'achats d'un client sur 6 mois : en janvier il achète un livre, en mars un stylo et un cahier, en juin un livre et une calculatrice. La séquence c'est l'ordre temporel de ces achats — c'est crucial, car acheter un stylo puis un cahier n'est pas pareil que l'inverse temporellement.

📅 Ordre temporel
→
🛍️ Événements

📊 Patterns séquentiels

Définitions formelles

Séquence

Une séquence s est une liste ordonnée d'événements (itemsets) : s = ⟨e₁ e₂ e₃ ... eₗ⟩

Chaque événement eᵢ est un ensemble d'items qui se produisent simultanément.

Exemple de cours

ID Séquence Notation Interprétation
σ₁ ⟨a(abc)(ac)d(cf)⟩ D'abord a, puis {a,b,c} ensemble, puis {a,c}, puis d, puis {c,f}
σ₂ ⟨(ad)c(bc)(ae)⟩ D'abord {a,d}, puis c, puis {b,c}, puis {a,e}
σ₃ ⟨(ef)(ab)(df)cb⟩ D'abord {e,f}, puis {a,b}, puis {d,f}, puis c, puis b
σ₄ ⟨eg(af)cbc⟩ D'abord e, puis g, puis {a,f}, puis c, puis b, puis c

Longueur d'une séquence

La longueur est le nombre total d'items (en comptant les répétitions).

Exemple: ⟨a(abc)(ac)d(cf)⟩ a longueur 9 (1+3+2+1+2)

Section 2: Sous-séquence et Support

Définition de sous-séquence

s' = ⟨e'₁...e'ₘ⟩ est une sous-séquence de s = ⟨e₁...eₙ⟩ si et seulement si :

Il existe des indices i₁ < i₂ < ... < iₘ tels que :

  • e'₁ ⊆ eᵢ₁
  • e'₂ ⊆ eᵢ₂
  • ... et ainsi de suite

Explication: Chaque événement de la sous-séquence doit être contenu (comme sous-ensemble) dans un événement de la séquence principale, EN PRÉSERVANT L'ORDRE.

Exemple

Séquence principale: ⟨a(abc)(ac)d(cf)⟩

Est-ce que ⟨(ab)c⟩ est une sous-séquence ?
Cherchons: {a,b} ⊆ ? puis c ⊆ ?
• {a,b} ⊆ (abc) ✓ (position 2)
• c ⊆ d ? NON, mais c ⊆ (cf) ✓ (position 5)
OUI, ⟨(ab)c⟩ est une sous-séquence

Support d'une séquence

Le support d'une séquence s est la fraction des séquences de la base de données qui contiennent s comme sous-séquence :

support(s) = |{σ ∈ DB | s ⊆ σ}| / |DB|

Exemple de calcul

Avec la BD de 4 séquences du cours :

Pattern recherché: ⟨a(b)⟩

• σ₁ = ⟨a(abc)(ac)d(cf)⟩ : a à position 1, b apparaît en position 2 → ✓
• σ₂ = ⟨(ad)c(bc)(ae)⟩ : a à position 1, b apparaît en position 3 → ✓
• σ₃ = ⟨(ef)(ab)(df)cb⟩ : a aparaît en position 2, b en même position → ✓
• σ₄ = ⟨eg(af)cbc⟩ : a en position 3, b en positions 4,5 → ✓

Support = 4/4 = 100% (très fréquent !)

Section 3: Algorithme GSP (Generalized Sequential Patterns)

GSP

Idée clé: Applique le paradigme Apriori aux séquences ! L'anti-monotonie du support fonctionne aussi pour les séquences : si une séquence est infréquente, alors toutes ses super-séquences le sont aussi.

Propriété d'anti-monotonie

Si support(s) < minSupp, alors pour toute super-séquence s'⊃s : support(s') < minSupp

Cela permet le pruning — élimination agressive des branches non-prometteuses.

Étapes de GSP

1 Trouver toutes les séquences fréquentes de longueur 1 (L₁)
2 Répéter jusqu'à aucune séquence fréquente trouvée :
a Générer les candidats Cₖ à partir de Lₖ₋₁
b Élaguer les candidats qui contiennent des sous-séquences infréquentes
c Scanner la BD pour compter le support de chaque candidat
d Garder les candidats avec support ≥ minSupp dans Lₖ

Fenêtres temporelles (Time windows)

Gaps et constraints

min_gap: Délai minimum entre deux événements consécutifs

max_gap: Délai maximum entre deux événements consécutifs

Window: Tous les items doivent apparaître dans un intervalle de temps W

Exemple: "⟨acheter chaussures puis détergent⟩ dans les 30 jours" — max_gap = 30 jours

Section 4: Algorithme SPADE (Sequential PAttern Discovery using Equivalence classes)

SPADE

Pivotement du paradigme: SPADE utilise le format vertical — au lieu de chercher les patterns dans les transactions, on cherche les transactions dans les patterns ! Les ID-lists (listes verticales) remplacent le format horizontal.

Format ID-list (Format vertical)

Représentation

Pour chaque séquence, on stocke : (seq_id, event_position)

Cela permet de calculer le support par intersection très rapide.

Exemple avec ID-lists

Pattern ⟨a⟩: {(1,1), (2,2), (3,2), (4,3)}
(dans séquence 1 à position 1, dans séquence 2 à position 2, etc.)

Pattern ⟨b⟩: {(1,2), (2,3), (3,2), (4,5)}

Pattern ⟨ab⟩ (a avant b):
Intersection avec condition a_pos < b_pos :
(1,1) et (1,2) → 1 < 2 ✓
(2,2) et (2,3) → 2 < 3 ✓
(3,2) et (3,2) → 2 < 2 ✗
(4,3) et (4,5) → 3 < 5 ✓

Support ⟨ab⟩ = 3/4 = 75%

Avantages de SPADE

Aspect Avantage
✅ Efficacité Les intersections d'ID-lists sont très rapides (opérations binaires)
✅ Scalabilité Peut traiter des gros datasets
✅ DFS Utilise Depth-First Search pour réduire la mémoire
❌ Complexité Plus complexe à implémenter que GSP

Section 5: Algorithme PrefixSpan (Prefix-projected Sequential Pattern)

PrefixSpan

Analogie Scrabble: PrefixSpan part d'un préfixe (début de séquence) et cherche comment il se prolonge. C'est comme compléter les mots dans un Scrabble — on sait qu'on commence par "CA", maintenant quels suffixes le complètent fréquemment ?

Concept clé: Projected Database

Définition

Pour un préfixe p donné, la projected database Dₚ contient :

Tous les suffixes des séquences de D qui commencent par p (après suppression du préfixe)

Exemple: Si p = ⟨a⟩ et on a une séquence ⟨a(bc)d⟩, le suffix est ⟨(bc)d⟩

Algorithme (schéma général)

1 Démarrer avec le préfixe vide (toute la BD)
2 Pour chaque préfixe p :
a Créer la BD projetée Dₚ
b Chercher les items fréquents dans Dₚ
c Pour chaque item fréquent i, créer nouveau préfixe p·i
d Récursivement appliquer l'algo au préfixe p·i

Section 6: Contraintes sur les Séquences

Comme pour les itemsets, on peut ajouter des contraintes pour canaliser la recherche de patterns séquentiels :

1. Contraintes de gap

max_gap: Délai maximum entre deux événements consécutifs

Exemple: "acheter pain et beurre dans les 7 jours"

2. Contraintes de fenêtre (Window)

window_size: Tous les items d'un événement doivent apparaître dans W jours

Exemple: "acheter au moins 3 items différents le même jour"

3. Contraintes d'items

Must-contain: Le pattern DOIT contenir certains items

Must-not-contain: Le pattern ne doit PAS contenir certains items

Exemple: "Pattern contenant 'pain' mais pas 'gâteau'"

4. Contraintes de longueur

min_length / max_length: Nombre minimum/maximum d'événements

Exemple: "Au moins 2 événements, max 5"

Trouver des sous-séquences communes dans l'ADN qui peuvent indiquer des maladies ou des mutations.

E-commerce

Section 8: Résumé et Comparaison des Algorithmes

Algorithme Paradigme Avantage principal Inconvénient
GSP Breadth-First Search (comme Apriori) Simple à comprendre et implémenter Peut être lent sur gros datasets
SPADE Vertical format (ID-lists) Très efficace, support rapide Plus complexe, consomme plus de mémoire
PrefixSpan Depth-First Search (Prefix projection) Efficace, utilise peu de mémoire Moins intuitif
À retenir

🎯 Objectif: Découvrir l'ordre temporel des événements qui se produisent fréquemment ensemble.

🔑 Clé: L'anti-monotonie du support permet l'élagage efficace.

💡 Choix d'algorithme:

  • GSP: petit dataset, compréhension pédagogique
  • SPADE: dataset moyen, besoin de vitesse
  • PrefixSpan: gros dataset, mémoire limitée
