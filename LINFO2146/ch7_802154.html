Chapitre 7 : IEEE 802.15.4

802.15.4 est le standard de base pour les réseaux IoT à faible puissance. C'est le protocole qui alimente ZigBee, 6LoWPAN, Thread et Bluetooth Low Energy (en partie). Comparé au WiFi, c'est 10x plus lent mais 10x moins consommateur de puissance.

1. Présentation 802.15.4

Le standard IoT

802.15.4 = "Wireless Personal Area Networks" (WPAN). C'est fait pour connecter des capteurs à bas débit, sur courte distance, avec batterie longue durée.

📡 Analogie : Si WiFi est l'autoroute à 100 km/h pour transférer des films en HD, 802.15.4 est la petite route à 5 km/h qui consomme 10x moins d'essence pour la même distance à pied. Idéal pour les capteurs qui envoient juste quelques bytes tous les 10 secondes.

2. Couche Physique (PHY)

Fréquences & Modulation

802.15.4 fonctionne sur trois bandes ISM (Industrial, Scientific, Medical) libres d'accès.

Bande Fréquence Canaux Largeur par canal Débit
European (868 MHz) 868.0 - 868.6 MHz 1 600 kHz 20 kbps (BPSK)
ISM 900 MHz 902-928 MHz (US) 10 2 MHz 40 kbps (BPSK)
2.4 GHz (WPAN) 2400 - 2483.5 MHz 16 5 MHz 250 kbps (O-QPSK)

Modulation : DSSS + O-QPSK

DSSS : Direct Sequence Spread Spectrum (aussi utilisé par CDMA). Chaque bit est étalé sur 32 chips.

O-QPSK : Offset Quadrature Phase Shift Keying. Deux voies en quadrature (cosinus et sinus) pour doubler le débit.

Résultat : 32 chips × 2 voies × 15.625 kchips/s = 250 kbps théorique. ✓

3. Topologies

Star vs Cluster-tree vs Mesh

IEEE 802.15.4 supporte trois topologies de réseau. Il est important de connaître leurs noms et caractéristiques pour l'examen !

⭐ Topologie Star

Un coordinateur PAN central (FFD). Tous les appareils (RFD ou FFD) communiquent uniquement avec le coordinateur.

  • Simple à mettre en place
  • Point unique de défaillance
  • Portée = portée du coordinateur
  • Ex: ZigBee light, BLE

🌳 Topologie Cluster-Tree

Arbre hiérarchique de coordinateurs FFD avec des feuilles RFD. Chaque coordinateur gère sa propre sous-zone.

  • Portée étendue (multi-hop)
  • Structure hiérarchique
  • Pas de boucles dans l'arbre
  • Utilisé dans ZigBee

🕸️ Peer-to-Peer (Mesh)

N'importe quel FFD peut communiquer avec n'importe quel autre FFD dans sa portée. Pas de hiérarchie.

  • Résilience maximale
  • Routage complexe (RPL)
  • Portée augmente avec les nœuds
  • Utilisé avec 6LoWPAN/RPL

4. Format de trame 802.15.4

Compacité

802.15.4 est extrêmement compact : la trame complète fait max 127 bytes ! 💪

Partie Signification Taille
PHR Physical Header (longueur de frame) 1 byte
MHR MAC Header (Frame Control, Seq, adresses) Jusqu'à 25 bytes
Payload Données utiles (ex: ZigBee, 6LoWPAN) Jusqu'à 102 bytes
MFR MAC Footer (FCS = Frame Check Sequence, CRC) 2 bytes

⚠️ Problème : IPv6 a un en-tête de 40 bytes minimum. Reste pour données = 102 - 25 - 40 = 37 bytes ! Donc IPv6 ne rentre pas bien. Solution : 6LoWPAN (Chapitre 9).

Adressage 802.15.4

Adresse courte (PAN ID + 16-bit address) : 2 + 2 = 4 bytes. Utilisée dans le même PAN.

Adresse longue (IEEE EUI-64) : 8 bytes. Adresse MAC globalement unique.

PAN ID : Personal Area Network ID (2 bytes). Identifie le réseau (exemple: ZigBee Home a un PAN ID spécifique).

5. CSMA/CA 802.15.4 (non-slotté)

MAC Layer

802.15.4 utilise CSMA/CA similaire à WiFi, mais avec des variantes adaptées au low-power.

CSMA/CA non-slotté 802.15.4

CCA (Clear Channel Assessment) : Écoute le canal.

Backoff aléatoire : Attend une durée aléatoire [0, CW] en unités de time slot.

Transmission : Envoie le frame si le canal reste libre.

ACK : Le destinataire envoie un ACK après un délai TAck.

Fenêtre de contention (CW) : Augmente en cas d'échec jusqu'à CW_max (par défaut : 4).

⏱️ Démo animée : CSMA/CA 802.15.4

✅ 802.15.4 CSMA/CA est plus simple que WiFi, avec CW plus petit (4 au lieu de 1023) pour réduire la latence et la consommation.

6. 🕐 Mode Slotté — Beacon-Enabled Mode

MAC Layer

Le mode slotté (beacon-enabled) ajoute une structure temporelle au réseau. Le coordinateur envoie régulièrement des BEACON frames pour annoncer la structure de la superframe.

Fonctionnement du Mode Slotté

BEACON frames : Le coordinateur PAN envoie des trames BEACON périodiquement pour synchroniser tous les nœuds et annoncer la structure de la superframe.

Structure de la Superframe :

  • Beacon : Annonce la structure (adresses actives, GTS en attente, etc.)
  • CAP (Contention Access Period) : 11 time slots. Tous les nœuds concourent via Slotted CSMA/CA.
  • CFP (Contention Free Period) : Jusqu'à 5 GTS (Guaranteed Time Slots). Slots TDMA réservés pour des nœuds spécifiques.
  • Periode Inactive : Tous les nœuds peuvent dormir pour économiser de l'énergie.

Beacon Interval (BI) : BI = aBaseSuperframeDuration × 2^BO, où BO = Beacon Order (0–14).

Superframe Duration (SD) : SD = aBaseSuperframeDuration × 2^SO, où SO = Superframe Order (0–14). Doit respecter SO ≤ BO.

CAP (Slotted CSMA/CA) : Tous les nœuds concourent normalement. Accès par contention.

CFP (Garantis TDMA) : GTS (Guaranteed Time Slots) réservés. Un nœud demande un GTS pendant le CAP → le coordinateur l'assigne dans le BEACON suivant.

Avantages : Latence prévisible (GTS), les nœuds peuvent dormir → bien meilleure efficacité énergétique.

Inconvénients : Nécessite une synchronisation stricte, le coordinateur est un point de défaillance unique.

Caractéristique Mode Non-Slotté Mode Slotté
Synchronisation Non requise Requise (BEACON)
Support GTS Non Oui (TDMA)
Écoute du récepteur Constante (toujours actif) Périodique (peut dormir)
Consommation énergétique Élevée Basse
Latence Variable (attente aléatoire) Prévisible (GTS)
✅ Le mode slotté est essentiel pour les applications IoT alimentées par batterie. Les nœuds synchronisés peuvent dormir lors de la periode inactive, réduisant drastiquement la consommation.

7. 📡 TSCH — Time Slotted Channel Hopping

MAC Layer

TSCH combine TDMA (Time Division Multiple Access) et frequency hopping (saut de fréquence) pour créer un réseau déterministe et resilient aux interférences.

Architecture TSCH

Slotframe : Un planning répétitif de timeslots. Chaque slot dure 10 ms. La bande 2.4 GHz contient 16 canaux.

Fonction de saut de fréquence :

channel = (ASN + channel_offset) mod num_channels

Où :

  • ASN : Absolute Slot Number (incrémente à chaque slot)
  • channel_offset : Assigné à chaque lien du planning
  • num_channels : Nombre de canaux disponibles (16 par défaut en 2.4 GHz)

Types de slots :

  • Dedicated slots : Réservés pour un lien spécifique (point-à-point).
  • Shared slots : Plusieurs nœuds concourent avec CSMA.

Approches de planification :

  • Static (Orchestra) : Planned à l'avance, pas de recalcul.
  • Centralized : Network manager calcule le planning global.
  • Decentralized : Algorithmes distribués (ex: TSCH-RPL).

Avantages : Diversité fréquentielle (une interférence sur un canal → utiliser un autre), efficacité TDMA, latence déterministe.

Utilisé dans : 6TiSCH (standard IETF combinant 6LoWPAN + TSCH + RPL), IoT industriel (usines, grilles intelligentes).

✅ TSCH apporte du déterminisme et de la résilience. Le saut de fréquence automatique évite les interférences stables. Parfait pour les applications critiques.

8. 🔒 Sécurité et Adressage IEEE 802.15.4

Security & Addressing

802.15.4 fournit deux types d'adresses et un chiffrement AES-128 optionnel au niveau MAC.

Sécurité 802.15.4

Chiffrement de trame : AES-128 optionnel dans l'en-tête MAC. Contrôlé par le champ Security Level.

Modes de sécurité :

  • None : Pas de sécurité.
  • MIC-32, MIC-64, MIC-128 : Message Integrity Code (intégrité uniquement, pas de confidentialité).
  • ENC : Chiffrement (confidentialité uniquement).
  • ENC-MIC : Chiffrement + intégrité (sécurité maximale).

Adressage

PAN ID : 16 bits. Identifie le Personal Area Network. Les nœuds dans des PAN différents ne communiquent pas directement.

Adresse courte (16 bits) : Assignée par le coordinateur, valide uniquement dans le PAN. Format: [PAN ID | 16-bit address].

Adresse longue (IEEE EUI-64, 64 bits) : Adresse MAC globalement unique. Format: [OUI | Identificateur nœud]. Utilisée pour l'authentification de sécurité.

Sélection du Coordinateur

Coordinateur : Le dispositif ayant la meilleure qualité de lien vers la racine/sink devient coordinateur.

✅ Combinaison d'adresses courtes (efficace en bande passante) et longues (globalement uniques). Sécurité AES optional mais critère pour ZigBee.

🧱 9. Protocoles construits sur IEEE 802.15.4

🧒 802.15.4 est la fondation, pas la maison complète !

IEEE 802.15.4 ne définit que les couches PHY et MAC. De nombreux protocoles de niveau supérieur (réseau, application) ont été construits par-dessus pour créer des systèmes IoT complets.

ProtocoleOrganismeUsage principalParticularités
ZigBee ZigBee Alliance Domotique, Smart Home Très populaire mais propriétaire (pas gratuit). Définit des couches réseau et application complexes.
Thread Thread Group Smart Home (Apple, Google) Peut aussi fonctionner sur Bluetooth et WiFi. Basé sur IPv6/6LoWPAN.
6LoWPAN + RPL IETF IoT générique, recherche Gratuit et open source. IPv6 sur 802.15.4. Utilisé dans Contiki-NG.
WirelessHART FieldComm Group Automatisation industrielle Réseau maillé industriel, haute fiabilité avec TDMA.
ISA100.11a ISA Automatisation industrielle Standard IEC pour les réseaux de terrain industriels sans fil.
MiWi Microchip Smart Home, alarmes Propriétaire Microchip. Moins de ressources que ZigBee (≈20 KB de code).
💡 À retenir : IEEE 802.15.4 est un standard de bas niveau. Les applications IoT nécessitent des protocoles de niveau supérieur (réseau, transport, application) construits par-dessus.

📅 10. Ordonnancement TSCH et multiplexage spatial

📘 Comment est déterminé l'ordonnancement TSCH ?

Le standard 802.15.4e ne spécifie pas comment l'ordonnancement doit être calculé — il définit seulement les mécanismes pour le distribuer et l'appliquer. Plusieurs approches existent :

  • Approche statique : l'ordonnancement est prédéterminé par l'opérateur réseau et programmé dans chaque nœud avant déploiement. Simple mais rigide.
  • Approche centralisée : les nœuds envoient leurs demandes de communication à un ordonnanceur central (ex. le coordinateur PAN) qui calcule l'ordonnancement optimal et le distribue à tous.
  • Approche décentralisée : les nœuds s'échangent leurs informations de demande et construisent l'ordonnancement collaborativement, sans entité centrale.
📘 Multiplexage spatial dans TSCH

TSCH suppose qu'un appareil ne peut envoyer ou recevoir que sur un canal à la fois. Cependant, si deux paires d'appareils sont suffisamment éloignées l'une de l'autre, l'ordonnanceur peut planifier l'utilisation du même canal au même moment pour ces deux paires simultanément — sans interférence.

Cela nécessite que l'ordonnanceur connaisse la position approximative des nœuds pour garantir qu'ils ne s'interfèrent pas mutuellement. C'est une forme de multiplexage spatial (SDMA) intégré dans TSCH.
