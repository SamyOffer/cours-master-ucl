Chapitre 5 : Accès au médium - Assignation fixe

Quand plusieurs appareils doivent partager le même canal de communication, il faut organiser l'accès. Cette section explore quatre approches d'assignation fixe : SDMA, FDMA, TDMA et CDMA.

1. SDMA (Space Division Multiple Access)

Concept clé

SDMA divise l'espace géographique. Chaque zone géographique a son propre canal.

💡 Analogie : Imaginez un grand bus avec plusieurs rangées. Chaque rangée est isolée et les conversations dedans ne gênent pas les autres rangées. SDMA c'est pareil : on divise l'espace en zones différentes.
✓ Avantages
  • Pas d'interférences entre zones
  • Utilisation efficace du spectre
✗ Inconvénients
  • Besoin d'énorme distance entre zones
  • Une station de base par appareil dans chaque zone
  • Très coûteux à déployer
🎯 SDMA est rarement utilisé en pratique à cause des distances énormes nécessaires et du coût d'infrastructure.

2. FDMA (Frequency Division Multiple Access)

Concept clé

FDMA divise le spectre de fréquence. Chaque appareil reçoit une bande de fréquence unique.

📻 Analogie : Les stations de radio FM. Chaque station (87.5-108MHz) a sa propre fréquence. Vous pouvez écouter France Inter (88MHz) ou NRJ (100.3MHz) simultanément sur le même appareil sans interférence.

3. TDMA (Time Division Multiple Access)

Concept clé

TDMA divise le temps. Chaque appareil reçoit des créneaux horaires (time slots) pour émettre/recevoir.

✈️ Analogie : À l'aéroport, les avions décollent par créneaux horaires (8h00, 8h15, 8h30, etc.). Un seul avion par créneau. C'est TDMA : chaque appareil a son créneau pour communiquer sur le même canal.

Formule d'efficacité TDMA

Symbole Signification Valeur typique
Tslot Durée du créneau horaire 0.577 ms (GSM)
Tguard Temps de garde (pour éviter les chevauchements) ~30 µs
Efficacité Proportion utilisable du créneau ~95% (GSM)

⏱️ Démo animée : TDMA - Partage du temps

Trois stations partagent le même canal en alternant leurs créneaux horaires :

Rouge : Station A | Bleu : Station B | Vert : Station C | Gris : Inactif

✅ TDMA est utilisé dans GSM (2G), qui a révolutionné la téléphonie mobile. Aussi utilisé dans certains systèmes IoT.

4. CDMA (Code Division Multiple Access)

Concept clé

CDMA utilise des codes pour distinguer les utilisateurs. Tous les appareils émettent simultanément sur la même fréquence, au même moment, mais avec des codes différents.

🎭 Analogie : Imaginez une pièce où 4 personnes parlent en même temps. Pour la personne A, elle écoute seulement les phrases en anglais (son "code"). Pour la personne B, elle écoute seulement les phrases en espagnol. Chacun ne capte que "son" code, même si tout le monde parle en même temps.

5. 📡 DSSS et étalement de spectre dans CDMA

Concept clé

CDMA fonctionne grâce à la technique d'étalement de spectre appelée DSSS (Direct Sequence Spread Spectrum). C'est le mécanisme physique qui permet à plusieurs utilisateurs de partager le même canal avec des codes différents.

🎵 Analogie : Imaginez une symphonie où chaque instrument joue la même note sur un très large bande de fréquences au lieu d'une seule fréquence précise. Le récepteur d'un instrument particulier applique un filtre pour récupérer seulement sa "note" même si tout le monde joue ensemble sur le même spectre large.

Processus DSSS - Étalement du signal

Étapes de la transmission :
  1. Signal original : bit de données (1 ou 0) avec période T
  2. Code PN (Pseudo-Noise) : séquence rapide de chips, chaque chip a une durée T_chip ≪ T
  3. Multiplication : chaque bit de données est multiplié par la séquence PN complète
  4. Signal étalé : un bit de données devient N chips (où N = T / T_chip)
  5. Transmission : le signal étalé est transmis sur un spectre large de bande passante

Désétalage à la réception

À la réception, le récepteur effectue une corrélation avec le même code PN :

Si le code PN est exact, la corrélation produit un pic (énergie concentrée). Si le code est incorrect, le résultat reste étalé (bruit).

Gain de traitement (Processing Gain) :

Plus le facteur d'étalement est grand, plus le gain de traitement est élevé, ce qui améliore la résistance aux interférences.

Symbole Signification Exemple

B_original Largeur de bande du signal originel 1 MHz (1 Mbit/s)
R_chip Débit de chips (chip rate) 10 Mchip/s
R_bit Débit de bits (bit rate) 1 Mbit/s
✓ Avantages de DSSS
  • Résistance aux interférences : le gain de traitement permet de rejeter les interférences en bande étroite
  • LPI (Low Probability of Intercept) : le signal étalé ressemble à du bruit pour un observateur non autorisé
  • Multiple access : plusieurs utilisateurs peuvent partager la même bande avec des codes différents
  • Anti-jamming : difficile de brouiller un signal large bande en utilisant des jammeuses en bande étroite
✗ Limitations de DSSS
  • Complexité du récepteur : la corrélation demande une puissance de calcul importante
  • Synchronisation : le récepteur doit être synchronisé avec le code PN (acquisition de code)
  • Consommation énergétique : opérations de corrélation et de synchronisation coûteuses
  • Utilisation du spectre : la bande passante réelle utilisée est large (10 MHz pour 802.11b à 1 Mbit/s)

Distinction : DSSS vs FHSS

DSSS (Direct Sequence) : le signal est étalé sur un spectre large et reste sur ce spectre pendant toute la transmission d'un bit. Un seul "canal large".

FHSS (Frequency Hopping Spread Spectrum) : le signal saute d'une fréquence à l'autre rapidement selon une séquence pseudo-aléatoire. Le spectre utilisé change constamment, mais à chaque instant, une seule fréquence est utilisée.

Comparaison :

  • DSSS : étalement instantané et continu → utilisation dense du spectre → plus rapide mais nécessite une réception plus précise
  • FHSS : saut de fréquence dans le temps → utilisation moins dense → plus lente mais meilleure résistance aux interférences localisées
  • DSSS : plus courant en WiFi, 3G, 802.15.4
  • FHSS : utilisé en Bluetooth, quelques standards militaires
✅ DSSS est le fondement physique de CDMA et de 802.11b WiFi. Le gain de traitement permet une coexistence de plusieurs utilisateurs sur la même bande. La complexité du décodage est le prix à payer pour cette flexibilité.

6. 📊 Conclusions et comparaison des méthodes d'accès fixe

Vue d'ensemble

Nous avons exploré quatre approches différentes pour l'accès au medium : SDMA, FDMA, TDMA et CDMA. Chacune a ses propres caractéristiques, avantages et limitations. Cette section propose une synthèse comparative.

Tableau comparatif : SDMA vs FDMA vs TDMA vs CDMA

OFDMA : Extension moderne de FDMA

OFDMA (Orthogonal Frequency Division Multiple Access) est une généralisation de FDMA qui combine les avantages de FDMA avec une meilleure gestion des ressources.

Principe : Au lieu de diviser la bande en larges sous-canaux (FDMA classique), OFDMA divise la bande en très nombreux sous-porteuses (subcarriers) étroites et orthogonales. Chaque utilisateur peut recevoir une allocation dynamique d'un ensemble de subcarriers.

L'orthogonalité entre les subcarriers élimine le besoin de guard bands (gain spectral majeur).

Standard Bande Subcarriers Cas d'usage
LTE (4G) 700 MHz - 2.6 GHz 72 - 1200 Telephonie mobile, données hauts débits
WiFi 6 (802.11ax) 2.4 GHz, 5 GHz, 6 GHz 242 - 996 Réseaux locaux sans fil
5G (NR) 0.6 GHz - 100 GHz 960 - 3600 Communications ultra-mobiles, IoT
DVB-T (Télévision numérique) 174 - 862 MHz 2048 - 8192 Diffusion vidéo terrestre
Avantages de OFDMA :
  • Efficacité spectrale : pas de guard bands → plus de subcarriers utiles
  • Flexibilité : chaque utilisateur peut avoir un nombre variable de subcarriers selon ses besoins
  • Robustesse au multipath : le préfixe cyclique (cyclic prefix) absorbe les échos de propagation
  • Gestion adaptative : allocation des subcarriers peut se faire dynamiquement selon la qualité du canal
Quand utiliser quelle méthode ?

SDMA : Réseaux cellulaires classiques avec stations de base. Peu utilisé seul, plutôt combiné avec FDMA/TDMA/CDMA.

FDMA : Systèmes de diffusion (radio FM), communication broadcast. Utilisé en 1G/2G cellulaire, LoRa.

TDMA : Systèmes avec besoins temporels prévisibles. GSM (2G), Time-Slotted Channel Hopping (TSCH) dans 802.15.4. Nécessite synchronisation horloge.

CDMA : 3G (WCDMA, CDMA2000). Offre grande capacité mais complexité et consommation énergétique plus élevées.

OFDMA : Standards modernes : 4G LTE, WiFi 6 (802.11ax), 5G NR. Excellent compromis flexibilité/efficacité. Demande équipements plus sophistiqués.

Cas spécial : réseaux IoT et accès aléatoire

Les méthodes d'accès fixe (SDMA, FDMA, TDMA, CDMA) supposent que l'ensemble des utilisateurs est connu a priori et que les ressources sont allouées statiquement.

Or, dans les réseaux 802.15.4 (Zigbee, Thread) et les réseaux de capteurs, le nombre d'appareils varie dynamiquement et imprévisiblement. Les appareils peuvent apparaître et disparaître à tout moment. Allouer des ressources fixes à chaque appareil serait inefficace et gaspillerait de la bande passante.

Solution : CSMA/CA (Carrier Sense Multiple Access with Collision Avoidance) — un protocole d'accès aléatoire et dynamique expliqué au chapitre 6.

  • Chaque appareil émet quand il a besoin, sans allocation préalable
  • Écoute d'abord le canal (Carrier Sense) pour éviter les collisions
  • Pas de synchronisation globale requise
  • Scalable : ajouter/retirer des appareils est transparent
✅ Résumé : SDMA, FDMA, TDMA et CDMA sont des techniques d'assignation fixe. OFDMA en est une évolution flexible. Pour les réseaux IoT dynamiques sans infrastructure, les méthodes aléatoires (CSMA/CA) sont préférées. Chaque approche répond à des besoins différents.

📐 Orthogonalité des codes CDMA — Théorie détaillée

En CDMA, les codes attribués aux différentes stations doivent être mutuellement orthogonaux. L'orthogonalité garantit que le récepteur peut extraire le signal d'un émetteur spécifique parmi tous les signaux superposés.

Définition mathématique de l'orthogonalité :
(a₀a₁…aₙ) · (b₀b₁…bₙ) = Σᵢ aᵢbᵢ = 0

Exemple de codes NON orthogonaux (problème) :

  • Station A : code (+1 +1)
  • Station B : code (-1 +1)
  • Produit : (+1)(-1) + (+1)(+1) = -1 + 1 = 0 → ces deux-là sont orthogonaux !

En pratique, les systèmes réels utilisent des codes plus longs que le minimum mathématique, créant une plus grande distance entre les codes. Cela rend le décodage plus robuste aux erreurs et permet de choisir des codes orthogonaux même si les émetteurs ne démarrent pas exactement au même moment (codes décalés de 1, 2, etc. pas).

⚠️ Problème near-far en CDMA

CDMA nécessite que tous les signaux arrivent avec à peu près la même puissance au récepteur. Si un émetteur A est beaucoup plus proche du récepteur que l'émetteur B :

  • Le signal de A arrive bien plus fort que celui de B
  • Lors de la corrélation pour extraire le signal de B, le signal dominant de A crée du bruit
  • → Le récepteur ne peut pas décoder B correctement

Solution dans les systèmes UMTS/3G : Le récepteur mesure la puissance reçue de chaque émetteur et envoie une information de retour (feedback) aux émetteurs, qui adaptent dynamiquement leur puissance d'émission. Ce mécanisme de contrôle de puissance en boucle fermée est essentiel au bon fonctionnement du CDMA.

💡 À retenir : Le problème near-far est l'un des principaux défis du CDMA. Sans contrôle de puissance, les nœuds proches du récepteur "noient" les signaux des nœuds éloignés.
