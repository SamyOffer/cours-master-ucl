Chapitre 15 : LTE — Mobilité et Gestion de Réseau

Maintenant que nous comprenons l'interface radio LTE, ce chapitre couvre comment gérer la mobilité (handover, paging, optimisation énergie DRX) et la signalisation réseau pour que les UEs restent connectés et localisés. C'est l'infrastructure "intelligente" derrière LTE.

1. MIB et System Information Blocks (SIBs)

Infos minimales

Avant tout appel/données, UE doit lire MIB et SIBs pour apprendre la configuration cellule.

MIB (Master Information Block)

Transmis sur PBCH (Physical Broadcast Channel) périodiquement (tous les 80 ms). Première info lue après synchronisation :

  • Largeur bande DL (1.4 / 3 / 5 / 10 / 15 / 20 MHz)
  • Nombre antennes eNodeB (1 / 2 / 4)
  • SFN (System Frame Number) : repère temporel
  • PHICH config : décide structure PHICH (ACK/NACK uplink)
  • PCFICH config : nombre symboles PDCCH (décide taille zone contrôle)

Taille MIB : 10 bits = très compact (contraste avec HTTP 700+ bytes !).

SIBs (System Information Blocks)

SIB Périodicité Contenu clé
SIB1 80 ms PLMN identity (MCC+MNC), Cell access info, Scheduling autres SIBs, Tracking Area Code.
SIB2 160 ms PRACH config (accès aléatoire), UL fréquence/bandwidth, Paging DRX cycle, infos downlink.
SIB3-13 Configurable Cellules voisines (handover), CSFB (circuit-switched fallback), WLAN, etc.

2. Tracking Areas (TA) et TAI

Localité granulaire

GSM/UMTS utilisaient LAC (Location Area Code). LTE utilise Tracking Area : une TA regroupe plusieurs cellules voisines.

🗺️ Analogie : Si chaque ville était une cellule, plusieurs villes formeraient un "pays" (TA). MME sait l'UE est "dans ce pays" sans connaître la ville exacte. Quand UE bouge vers autre pays : mise à jour (coûte signalisation), mais rare.
  • TAC (Tracking Area Code) : ID local dans le réseau. 16 bits.
  • TAI (Tracking Area Identifier) : Global = PLMN ID (MCC + MNC) + TAC. Utilisé pour roaming international.
  • Typical size : 1 TA = 5-50 cellules (dépend densité), zone quelques km².

3. MME (Mobility Management Entity)

Cerveau du cœur réseau

Le MME (Mobility Management Entity) remplace l'MSC de GSM. C'est le cerveau du cœur réseau LTE pour la mobilité et la sécurité.

Responsabilités MME

  • Localisation UE : Sait quelle TA (et donc quelle eNodeB) a chaque UE.
  • Authentification + Sécurité : Dialogue avec HSS pour authentifier UE (RAND/SRES), configurer chiffrement NAS.
  • Gestion sessions : Crée/modifie/supprime GTP tunnels pour bearers vers SGW (Serving Gateway).
  • Handover : Si X2 insuffisant (eNodeBs lointains), MME coordonne handover.
  • Paging : Quand appel entrant → MME page UE sur toutes eNodeBs de la TA.
  • Redondance : Plusieurs MMEs → GUMMEI (Global Unique MME ID) = PLMN ID + MMEGI (group) + MMEC (code).

Identités UE

Identité Quand utilisée Validité
IMSI Première connexion au réseau. Stockée sur SIM. Permanente (international)
GUTI Après authentification. Envoyé chiffré. Valide dans MME/groupe MMEs
s-TMSI Raccourci : MMEC + M-TMSI. Utilisé localement. Valide dans groupe MMEs

Pourquoi plusieurs identités ? Protection vie privée. IMSI envoyé en clair risque qu'attaquant sache numéro = appelable. GUTI chiffré => protection. Changement GUTI périodiquement = empêche tracking simple.

4. Paging — Rester localisé et économe

Réveille-matin du téléphone

Active UE : MME connaît la cell exacte → push direct. Idle UE (endormi) : MME sait seulement la TA → paging sur toutes cells de la TA.

Paging quand appel entrant

  1. Appel arrive → MSC/HSS : Appel VoLTE pour UE. Network contacte MME : "Page cet UE".
  2. MME envoie Paging Request → toutes eNodeBs de la TA (sur S1-MME).
  3. eNodeBs diffusent sur PCH (Paging Channel) : "UE X, répondez !".
  4. UE en RRC Idle réveille : Tous les N×100ms (typiquement N=128 → toutes les 12.8 sec), UE réveille et écoute PDCCH.
  5. UE détecte son I-RNTI : PDCCH contient "paging" pour lui → UE se réveille complètement.
  6. UE envoie Paging Response sur PRACH → BSS → MSC. Connexion établie.

5. DRX (Discontinuous Reception) — Mode économie d'énergie

Sommeil intelligent

DRX : Au lieu d'écouter 24/7, UE dort et se réveille périodiquement. Économise batterie drastiquement.

🔋 Analogie : Sans DRX = téléphone toujours en écouteur. Avec DRX = dormir, se réveiller toutes les 12 secondes, vérifier si message. Batterie dure 10x plus longtemps.

C-DRX (Connected Mode DRX)

UE en connexion RRC, mais données intermittentes :

  • Awake mode : Écoute PDCCH chaque sous-frame (activité détectée).
  • Inactivity Timer expires (ex: 1 sec sans data) → Short DRX.
  • Short DRX Cycle : ~20 ms sleep, se réveille tous les 20 ms (configurable). Acceptable pour réactivité.
  • Short DRX Timer expires (ex: 5 sec en short DRX) → Long DRX.
  • Long DRX Cycle : Très long sleep (ex: 2.56 s), se réveille tous les 2.56 s. Batterie excellent.

Paramètres : Envoyés par eNodeB via RRC dans ConnectionReconfiguration/ConnectionSetup sur DCCH.

DRX Idle Mode

UE en RRC Idle (pas de data) :

  • RRC Inactivity Timer : Après X ms sans données, eNodeB envoie RRC Release → UE va en Idle.
  • Paging DRX Cycle : Très long (ex: 2.56 s). UE écoute toutes les 2.56 s pour appels entrants.
  • Paramètres : defaultPagingCycle dans SIB2.

DRX Timing Advance

⏱️ Timing critère : UE doit se réveiller EXACTEMENT au moment où eNodeB envoie paging. Décalage = rate la paging = perte d'appel. RTC (Real Time Clock) du téléphone doit être précis. Dérive RTC ≤ 100 ppm = ok.

6. Information System Broadcast (continuité)

Configuration cellule

MIB+SIB1+SIB2 sont obligatoires. Autres SIBs optionnels mais annoncés dans SIB1.

Stratégie de transmission

  • MIB : PBCH, tous les 80 ms, très fiable (4 répétitions).
  • SIB1 : PDSCH, tous les 80 ms, SIB1 change rarement (configuration basique).
  • SIB2 : PDSCH, tous les 160 ms. Contient PRACH/paging/UL config.
  • Autres SIBs : Scheduled as needed. Ex: SIB3 (neighbor cells) pour handover.

Avantage : UE peut démarrer rapide (lire MIB + SIB1 + SIB2 = ~320 ms). Pas besoin attendre tous les SIBs si unnecessary.

7. S1-AP — Signalisation eNodeB ↔ MME

Protocole contrôle réseau

S1-AP (S1 Application Protocol) : signalisation entre eNodeB et MME sur S1-MME interface.

  • Transport : SCTP (Stream Control Transmission Protocol, orienté message fiable).
  • Usages majeurs :
    • Initial UE Message : UE → eNodeB → S1-AP → MME (attach request, context setup).
    • Authentication : MME ↔ HSS via DIAMETER, puis MME → eNodeB setup sécurité.
    • Downlink NAS Transport : NAS PDUs (conteneur haut niveau) transportés via S1-AP.
    • Paging : MME → eNodeB : "page cet UE".
    • Handover : eNodeB A → MME → eNodeB B (coordination).

NAS (Non-Access Stratum)

NAS = protocole haut niveau entre UE et MME (non radio). Contient :

  • Mobility management : Attach, detach, TA update, authentication.
  • Session management : PDN connectivity requests (allocate IP), bearer setup.
  • Signaling : Chiffré et intégré au RRC (transporté sur SRBs radio).

Distinction importante : RAS (Access Stratum) = radio (RRC/MAC/PHY). NAS = signalisation réseau haut niveau (non radio).

8. Handover (Mobilité cellule-à-cellule)

Rester connecté en mouvement

Quand UE bougé d'une cellule à une autre, LTE doit switcher vers la meilleure cellule sans casser la connexion.

Handover X2

Scénario : eNodeB A, UE se rapproche de eNodeB B :

  1. eNodeB A mesure signal voisin : UE envoie measurement report (cellule B devient meilleure).
  2. eNodeB A → eNodeB B (X2 interface) : Handover request. B prépare ressources.
  3. eNodeB B alloue RBs, prépare C-RNTI (Cell RNTI), envoie ACK à A.
  4. eNodeB A → UE : Handover command (RRC) + "switch to eNodeB B, new C-RNTI".
  5. UE synchronise sur eNodeB B, envoie Handover Confirm sur PRACH.
  6. eNodeB B → MME (S1-AP) : Path switch request (change tunnels de A à B).
  7. eNodeB A transfère data en vol à B (X2-U), puis se déconnecte.

9. Measurement et Reporting — Décisions handover

Mécanisme décision

UE mesure signaux et rapporte à eNodeB. eNodeB décide de handover sur base rapports.

Mesures UE

  • Serving cell RSRP (Reference Signal Received Power) : Force signal reçue. Typiquement -75 à -140 dBm (+ fort = meilleur).
  • Neighbor cell RSRP : UE peut mesurer jusqu'à 8 voisins (SIB3 liste candidates).
  • RSRQ (Reference Signal Received Quality) : Force + interférence = qualité vraie.
  • Measurement Report : UE envoie périodiquement (configurable, ex: tous les 100 ms). Contient RSRP/RSRQ cellule serving + voisins.

Trigger handover

eNodeB décide handover via hysteresis + timing :

  • Hysteresis : Ex, basculer si voisin RSRP > serving RSRP + 3 dB (évite ping-pong).
  • Time-to-Trigger : Ex, 256 ms = condition doit tenir 256 ms avant handover (smooth variations).
  • Paramètres NAS : Hysteresis, TTT, measurement bandwidth envoyés via RRC.

10. Roaming International et SSS7 vs S1-AP

Quand UE va à l'étranger

UE détecte PLMN étranger, s'attache à réseau local. Cœur réseau doit trouver HLR "home" dans pays origine.

Architecture roaming :

  • UE en France, home HSS en Belgique.
  • UE s'attache au réseau France.
  • eNodeB France → MME France → G-MME France.
  • G-MME France → S6a interface (DIAMETER) → HSS Belgique (home).
  • HSS Belgique authentifie UE France, retourne profil.
  • Signalisation revient à MME France, qui établit session.

S6a vs SS7 : GSM utilisait SS7 (Signaling System #7, protocol télécom classique). LTE utilise DIAMETER sur S6a (plus moderne, basé IP). Mais roaming SS7/S6a backwards compatible.

Résumé pour l'examen

Points-clés mobilité et réseau LTE :
  • MIB : Lue en 1er, PBCH, 10 bits. Contient bandwidth, antennes, SFN, PHICH/PCFICH config.
  • SIBs : SIB1 (PLMN/TAC), SIB2 (PRACH/DRX/UL), SIB3+ (voisins, autres configs).
  • Tracking Area : Groupe cellules. TAC local, TAI global (+ PLMN). UE sait TA quand détection paging.
  • MME : Cœur mobilité. Auth (HSS), gestion sessions (SGW), paging, handover. Redondance = GUMMEI.
  • Identités UE : IMSI (permanent), GUTI (session chiffré), s-TMSI (local). Privé.
  • Paging : Active UE localisée à cell. Idle UE localisée à TA. Tous les N×100ms réveil, écoute PDCCH.
  • DRX : C-DRX (connected) = short/long cycles. Idle mode = paging DRX. Économie batterie majeure.
  • S1-AP : Signalisation eNodeB ↔ MME. NAS transporté. SCTP, fiable.
  • Handover X2 : eNodeB A → eNodeB B (direct via X2), rapide, sans MME si possible. Mesure + hysteresis + TTT.
  • Measurement Report : UE → eNodeB (RSRP, RSRQ). Décision handover. SIB3 liste voisins.
  • Roaming : HSS home distant. S6a (DIAMETER) pour auth internationale. Architecture SSS7 hérité.

Fluxes clés à retenir

  • Attach initial : UE PRACH → eNodeB S1-AP InitialUEMessage (+ NAS Attach Request) → MME authentifie (HSS) → alloue GUMMEI/GUTI → SGW alloue IP → Data + signaling bearers setup.
  • Appel entrant : Réseau → G-MME → MME (trouve TA) → eNodeBs paging PCH → UE en Idle réveille (PDCCH) → répond PRACH → connexion RRC → signaling volte (SIB2 contient PRACH config).
  • Handover en mouvement : eNodeB A reçoit measurement report → détecte voisin meilleur → X2 handover request → eNodeB B prépare → UE RRC command → UE synchro B → handover confirm → X2-U data forward → S1-AP path switch.
Résumé ultime pour l'examen lundi : LTE = rupture (OFDMA+SC-FDMA, IP-only, eNodeB applaini). Interface radio chapitre 14. Mobilité/réseau chapitre 15 = MME cerveau + S1-AP signalisation + DRX économie + handover X2 rapide. GSM/GPRS/UMTS anciens mais posent fondations (HLR→HSS, TDMA→OFDMA, SGSN→SGSN remplacée par SGW). Comprendre architecture = maîtriser LTE.
