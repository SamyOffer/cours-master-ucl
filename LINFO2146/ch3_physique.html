🌊 1. Ondes électromagnétiques

🧒 C'est quoi une onde radio ?

Imagine jeter un caillou dans l'eau : des cercles se propagent autour du point d'impact. Une onde radio c'est pareil, mais invisible — c'est une oscillation du champ électromagnétique qui se propage dans l'espace à la vitesse de la lumière (3×10⁸ m/s).

Ton téléphone, le WiFi, la radio FM — tout ça utilise des ondes électromagnétiques à différentes fréquences pour transporter des informations.

📊 2. Analyse de Fourier — Décomposition en fréquences

🧒 C'est quoi la décomposition de Fourier ?

Imagine un accord de piano avec plusieurs touches enfoncées en même temps : tu entends plusieurs fréquences superposées. La décomposition de Fourier dit : tout signal complexe peut être vu comme une somme de sinus simples à différentes fréquences.

En radio, plus tu veux envoyer de données vite, plus le signal contient de hautes fréquences. L'électronique + l'antenne doivent être optimisées pour ces fréquences.

📌 En une phrase : tout signal s(t) peut s'écrire comme une somme d'ondes sinusoïdales de fréquences, phases et amplitudes différentes (théorème de Fourier).

🎛️ 3. Modulation — Encoder l'information dans une onde

🧒 C'est quoi la modulation ?

Imagine une lampe de poche. Pour transmettre un message en morse, tu l'allumes et l'éteins selon un code. La lumière c'est la porteuse — son intensité est modulée pour porter l'information.

En radio, on module l'Amplitude (ASK/AM), la Fréquence (FSK/FM), ou la Phase (PSK/PM) d'une onde porteuse pour transporter des bits.

🔐 4. Codage de canal — Protection contre les erreurs

🧒 C'est quoi le codage de canal ?

Imagine envoyer une lettre dans un pays où les facteurs perdent parfois des pages. Tu envoies des copies de chaque page (redondance) pour que le destinataire puisse reconstruire le texte même si certaines pages sont perdues.

  • Bits de parité : un bit supplémentaire pour détecter 1 erreur
  • CRC (Cyclic Redundancy Check) : détecte plusieurs erreurs
  • Block coding : ajoute des bits de correction à des blocs de données
Le codage de canal ajoute de la redondance pour permettre au récepteur de détecter et corriger des erreurs de transmission.

L'interleaving — Contre les erreurs en rafale (burst errors)

🧒 Le problème des burst errors

Un parasite électrique peut corrompre plusieurs bits consécutifs (erreur en rafale). Si tous ces bits sont dans le même mot de code, le code correcteur ne peut pas les corriger.

Solution — Interleaving : on entremêle les données avant transmission. Ex : "Hello" "World" → "HWeol" "rllod". Une rafale n'affecte alors qu'un bit par mot de code.

Interleaving = mélanger les données pour distribuer les erreurs en rafale sur plusieurs mots de code différents → chaque mot n'a qu'une seule erreur → corrigeable.

📡 5. DSSS — Étalement de spectre

🧒 C'est quoi le DSSS ?

Au lieu d'envoyer chaque bit directement, on le multiplie par une séquence de code (des ±1 appelés "chips"). Le signal résultant est beaucoup plus large en fréquence mais plus bas en amplitude → moins sensible aux interférences étroites.

DSSS = Direct-Sequence Spread Spectrum. Utilisé dans IEEE 802.11b et IEEE 802.15.4.

Le récepteur qui connaît le code peut retrouver le bit original par corrélation. Les interférences qui ne connaissent pas le code apparaissent comme du bruit faible.
📌 Exemple numérique du cours

Émetteur A envoie bit "1". Code A : (-1, +1, -1, -1, +1, +1). Signal A = (+1)×code = (-1,+1,-1,-1,+1,+1)

Émetteur B envoie bit "0" (→ -1). Code B : (+1,+1,-1,+1,-1,+1). Signal B = (-1,-1,+1,-1,+1,-1)

Signal reçu (superposition) : (-2, 0, 0, -2, +2, 0)

Décodage A : (-2,0,0,-2,+2,0) · (-1,+1,-1,-1,+1,+1) = 2+0+0+2+2+0 = +6 > 0 → bit "1" ✅

🫥 Le problème de la station cachée (Hidden Terminal Problem)

🧒 Pourquoi CSMA ne suffit pas en WiFi/IoT ?

En réseau filaire, si deux câbles transmettent en même temps, on détecte la collision électriquement. En sans-fil, c'est plus compliqué : un nœud peut ne pas entendre un autre nœud qui est pourtant en train d'émettre !

La station cachée = un nœud qui ne peut pas entendre un autre nœud concurrent, mais dont les signaux créent quand même une collision chez le destinataire commun.

Scénario avec 3 stations équidistantes A, B, C

A ━━━━━ B ━━━━━ C
✅ Dans la portée de B :
A peut entendre B
C peut entendre B
❌ Hors portée :
A ne peut PAS entendre C
C ne peut PAS entendre A
📌 Déroulement du problème
  1. A commence à transmettre vers B
  2. C veut aussi transmettre. C écoute le canal → il semble libre (C n'entend pas A)
  3. C commence à transmettre vers B en même temps que A
  4. Collision chez B : B reçoit les deux signaux mélangés → trame corrompue
  5. Ni A ni C ne détectent la collision (ils ne s'entendent pas)

Conséquence : CSMA/CD ne fonctionne pas en sans-fil — on ne peut pas toujours "entendre" si le canal est vraiment libre.

Solution : RTS/CTS

ÉtapeActionQui agit
1Envoie RTS (Request To Send) à B après que le canal est libre depuis IFSA → B
2Répond avec CTS (Clear To Send) à A (après SIFS)B → A (diffusé)
3Entend le CTS de B → sait que B est occupé → attendC (entend le CTS)
4Commence à envoyer ses données, en toute sécuritéA → B

📶 Modèles de réseau — OSI et TCP/IP dans les réseaux IoT

🧒 Pourquoi plusieurs couches ?

Imagine construire une maison : tu as besoin de fondations solides, puis un cadre, puis des murs, puis l'électricité, puis la décoration. Chaque étage repose sur celui du dessous. Un réseau c'est pareil ! Les modèles en couches divisent la communication en étages, où chaque couche a un rôle précis et ne doit pas se soucier des détails des autres couches.

Modèle OSI (7 couches)

Le modèle OSI (Open Systems Interconnection) est un standard international qui décrit comment les systèmes informatiques communiquent. Il a 7 couches :

  • 7. Application (HTTP, SMTP, DNS, CoAP, MQTT)
  • 6. Présentation (compression, chiffrement, traduction de formats)
  • 5. Session (établissement et gestion de sessions)
  • 4. Transport (TCP, UDP, fiabilité end-to-end)
  • 3. Réseau (routage, adressage IP)
  • 2. Liaison de données (MAC, encadrage, accès au médium)
  • 1. Physique (transmission des bits, modulation, antennes)

Modèle TCP/IP simplifié (5 couches) — le modèle IoT

En pratique IoT, on utilise un modèle simplifié à 5 couches :

📌 Pile TCP/IP pour IoT

Couche 5 : Application

  • CoAP (Constrained Application Protocol) : léger, sans état, pour capteurs avec peu de ressources. Port 5683 UDP.
  • MQTT (Message Queuing Telemetry Transport) : pub/sub, excellent pour IoT. Port 1883 TCP.
  • HTTP : lourd, peu efficace pour batterie, mais utilisé en rétroréseaux.

Couche 4 : Transport

  • UDP : sans connexion, pas de handshake, léger (bon pour batterie). Pas de garantie de livraison.
  • TCP : avec connexion, fiable mais lourd (handshake, accusé de réception, buffers). Éviter en IoT batterie.

Couche 3 : Réseau

  • IPv6 : adressage universel, stateless autoconfiguration (parfait IoT).
  • RPL (Routing Protocol for Low-power and Lossy networks) : routage par arborescence, efficace en énergie.

Couche 2 : MAC / Liaison

  • Encadrage : découpe données en trames, ajoute adresses MAC source/destination.
  • CSMA/CA (IEEE 802.15.4) : écoute le canal, attend si occupé, puis envoie (collision évitée).
  • TDMA : chaque nœud a son créneau horaire (déterministe, très efficace).

Couche 1 : Physique

  • IEEE 802.15.4 (2.4 GHz, ±80 kHz BW, ~250 kbps) : standard pour Zigbee, 6LoWPAN.
  • LoRa (868/915 MHz, spread-spectrum, portée km) : pour données rares, très basse puissance.

Pourquoi UDP plutôt que TCP en IoT ?

TCP: Handshake 3-way, acknowledgements, congestion control, retransmissions → beaucoup de paquets même pour peu de données. Idéal web/vidéo où chaque octet compte.

UDP: Envoie juste les données, aucun établissement. Le protocole application (CoAP) ajoute la fiabilité optionnelle si nécessaire → très efficace énergie.

En IoT, on préfère UDP à TCP car TCP est trop lourd (handshake, congestion control, buffers). CoAP ajoute la fiabilité au niveau applicatif si nécessaire.

📻 Types de réseaux sans fil — du BAN au WAN

🧒 Combien loin je peux envoyer ma donnée ?

Les réseaux sans fil ne sont pas tous identiques. Certains sont minuscules (bracelet connecté qui parle à ta montre, 1 mètre), d'autres énormes (satellite qui couvre un pays, 1000+ km). On classifie les réseaux par zone de couverture (range).

Les 5 types de réseaux (du plus proche au plus lointain)

📌 Hiérarchie de couverture

🔹 BAN (Body Area Network) : <1 mètre

  • Capteurs sur/autour du corps (fréquence cardiaque, température).
  • Technologies : NFC (<10 cm), Bluetooth LE (10 m, mais data rate bas = efficace batterie).
  • Exemple : bracelet fitness ↔ téléphone.

🔹 PAN (Personal Area Network) : ~10–100 mètres

  • Domicile, bureau, campus réduit. C'est le réseau favori d'IoT WSN.
  • Technologies : IEEE 802.15.4 (Zigbee, 6LoWPAN), Bluetooth Classic, Z-Wave.
  • Data rate : 250 kbps (802.15.4), puissance basse, batterie = mois/ans.
  • Exemple : capteurs température dans appart → hub central.

🔹 LAN (Local Area Network) : ~100 m–1 km

  • Bâtiment, étage, usine, campus.
  • Technologies : WiFi 802.11 (2.4/5/6 GHz), Ethernet câblé.
  • Data rate : 50–1000+ Mbps, consommation plus haute (WiFi = batterie ~heures).
  • Exemple : caméra IP, imprimante réseau.

🔹 MAN (Metropolitan Area Network) : ~1–10 km

  • Ville, campus large, parc industriel.
  • Technologies : WiMAX (maintenant obsolète), certains déploiements LoRa municipaux.
  • Intermédiaire entre LAN efficace et WAN long-range.

🔹 WAN (Wide Area Network) : >10 km

  • Région, pays, globe. Réseau décentralisé, infrastructure lourde (base stations, satellites).
  • Technologies : LoRaWAN (>30 km rural, ~1–5 km urbain, bande libre 868 MHz), Sigfox (même bande, ultra-étroit), LTE/NB-IoT (réseau cellulaire), 5G, Satellites.
  • Data rate : LoRa 50 bps–11 kbps (très bas), NB-IoT ~250 kbps, batterie mois/ans.
  • Exemple : capteur de qualité d'air urbain → cloud, station météo rurale.

Architecture typique : IoT WSN avec backhaul WAN

Un réseau de capteurs IoT classique fonctionne ainsi :

  • Les nœuds capteurs communiquent en PAN (IEEE 802.15.4, Zigbee) : efficace, économe en batterie, portée locale.
  • Un ou plusieurs routeurs/répéteurs relient le PAN à l'extérieur (réseau maillé, RPL).
  • La passerelle (gateway) se connecte au WAN (LoRaWAN, NB-IoT, WiFi) pour envoyer les données au cloud.

Exemple concret : capteurs de température dans une maison (PAN 802.15.4) → box domotique (gateway WiFi) → serveur cloud (WAN).

En IoT, on préfère un réseau local PAN pour la communication courte-portée (énergie efficace), relié par un WAN robuste pour le backhaul vers le cloud.

📡 Démodulation — Défis côté récepteur

🧒 Recevoir un signal, c'est compliqué !

Envoyer un signal modulé est une chose. Le démoduler correctement côté récepteur est une tout autre affaire. Plusieurs défis se posent simultanément.

📘 Théorie : Défis de la démodulation
  • Synchronisation de bits : quand commence exactement un symbole ou un bit ? Le récepteur doit trouver les frontières temporelles précises.
  • Synchronisation de trames : quand commence un paquet/trame dans le flux de bits reçu ?
  • Dérive de fréquence : l'émetteur et le récepteur n'écoutent pas exactement sur la même fréquence (à cause du vieillissement électronique, des variations de température…)
  • Bruit et interférences : le canal radio ajoute du bruit qui peut faire confondre un "0" avec un "1"

Ces problèmes sont résolus par une combinaison de matériel spécialisé et d'algorithmes de traitement du signal — sujet du domaine des télécommunications, pas du cours LINFO2146.

La démodulation est le processus inverse de la modulation : le récepteur doit retrouver les bits originaux à partir du signal reçu bruité.

📏 Capacité maximale d'un canal — Nyquist & Shannon

📘 Théorie : Débit maximal (Nyquist)

Dans des conditions idéales (sans bruit), le débit binaire maximal d'un canal de bande passante B utilisant une modulation à m niveaux d'amplitude est :

Débit max = 2 × Bande passante × log₂(m)

Exemples :

  • ASK simple (2 niveaux, m=2) : Débit max = 2 × B × 1 = 2B
  • QAM-16 (16 symboles, m=16) : Débit max = 2 × B × 4 = 8B
  • QAM-64 (64 symboles) : Débit max = 2 × B × 6 = 12B

En pratique, on ne peut pas utiliser des m très grands à cause du bruit — les symboles se confondent. Le choix de la modulation est un compromis entre efficacité spectrale (bits/Hz) et robustesse aux erreurs.

Plus m est grand, plus on transmet de bits par symbole, mais plus le signal est sensible au bruit. WiFi 6 utilise QAM-1024 (10 bits/symbole) dans des conditions de signal excellentes.
