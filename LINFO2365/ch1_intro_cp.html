🧩 Qu'est-ce que la Programmation par Contraintes ?

🧒 C'est quoi la PC ?

Imagine que tu dois placer des rois d'échecs sur un plateau sans qu'ils se mangent mutuellement. Tu pourrais essayer toutes les combinaisons possibles — mais ça prendrait une éternité. La Programmation par Contraintes (CP), c'est une façon de dire "voilà les règles du jeu", et laisser l'ordinateur trouver une solution intelligemment.

Analogie : C'est comme résoudre un Sudoku. Tu n'essaies pas toutes les combinaisons — tu utilises les règles (chaque ligne, colonne, région doit avoir 1-9 exactement une fois) pour éliminer des chiffres impossibles, jusqu'à trouver la solution.

💡 À retenir : En CP, tu décris le problème (variables + contraintes), et le solveur trouve la solution. C'est déclaratif.
📦

Variables

Ce qu'on cherche à déterminer. Ex : la colonne de chaque reine

🎯

Domaines

Les valeurs possibles pour chaque variable. Ex : {1, 2, 3, ..., N}

📏

Contraintes

Les règles à respecter. Ex : pas deux reines sur la même ligne

🔍

Solveur

L'algorithme qui explore les possibilités et trouve une solution

Définition formelle d'un CSP : Un problème de satisfaction de contraintes est un triplet ⟨X, D, C⟩ où :

• X = ensemble de variables {x₁, x₂, ..., xₙ}

• D = ensemble de domaines {D₁, D₂, ..., Dₙ} (chaque Dᵢ = valeurs possibles pour xᵢ)

• C = ensemble de contraintes (relations sur les variables)

Une solution est une affectation de valeurs aux variables qui satisfait toutes les contraintes.

♟️ Le Problème des N-Reines

🧒 C'est quoi le problème des N-Reines ?

Place N reines sur un échiquier N×N de façon à ce qu'aucune reine ne menace une autre. En échecs, une reine attaque toutes les cases de sa rangée, sa colonne et ses diagonales.

Modélisation intelligente : Au lieu d'un booléen par case (N² variables), on utilise une variable par colonne indiquant sur quelle rangée se trouve la reine de cette colonne. On a ainsi seulement N variables, chacune avec un domaine {0, ..., N-1}.

💡 Le bon modèle réduit drastiquement la taille de l'espace de recherche !
📐 Formules

📌 "Deux reines ne se menacent pas si elles sont sur des rangées différentes et des diagonales différentes"

SymboleSignification
qᵢLa rangée de la reine dans la colonne i
i, jIndices de colonne (0 à N-1)
|qᵢ - qⱼ|Différence de rangées (valeur absolue)
|i - j|Différence de colonnes

Exemple (N=4) : Si q₀=1, q₁=3 : |1-3|=2, |0-1|=1 → 2 ≠ 1 ✅ pas en diagonale.

🌳 Recherche DFS + Filtrage

🧒 C'est quoi DFS ?

DFS = Depth-First Search = recherche en profondeur d'abord. Imagine un labyrinthe : tu choisis un chemin, tu avances le plus loin possible, et si tu te retrouves bloqué, tu reviens sur tes pas (backtrack) et tu essaies un autre chemin.

En CP, on construit un arbre de recherche : à chaque nœud, on choisit une variable et une valeur. Si une contradiction apparaît, on revient en arrière.

Le filtrage (ou propagation) : avant de continuer, on élimine les valeurs impossibles dans les domaines. C'est ce qui rend CP puissant — on taille l'arbre avant même d'explorer.

💡 DFS seul = force brute. DFS + filtrage = exploration intelligente.

  • Choisir une variable non encore fixée (par ex. la première avec le plus petit domaine)

  • Essayer une valeur de son domaine — créer un nœud dans l'arbre

  • Propager : éliminer les valeurs impossibles dans les autres domaines

  • Si échec (domaine vide) → backtrack : défaire les choix et essayer une autre valeur

  • Si tous les domaines sont des singletons → solution trouvée !

⚙️ TinyCSP — Rendre CP Générique

🧒 C'est quoi TinyCSP ?

Au lieu d'écrire du code spécialisé pour chaque problème (N-Reines, Sudoku, coloration de graphe...), on crée une architecture réutilisable : TinyCSP. C'est un mini-solveur CP avec des abstractions claires.

Analogie : C'est comme créer une machine à résoudre les puzzles en général, plutôt qu'une machine spécialisée uniquement pour le Sudoku.

💡 TinyCSP pose les bases de MiniCP, le vrai solveur du cours.
📦

Variable

Objet avec un domaine modifiable. Supporte remove(v), fix(v), isFixed()

📋

Constraint

Interface avec post() et propagate(). post() = s'abonner aux événements

⚡

Solver

Gère les contraintes, déclenche la propagation, et le backtrack

🌳

DFSearch

Construit l'arbre de recherche, gère les branchements

Paradigme déclaratif avec TinyCSP :

• L'utilisateur déclare variables, domaines et contraintes

• Le solveur gère tout le reste (propagation, backtrack, recherche)

• Les contraintes sont réutilisables d'un problème à l'autre

• Le code CP pour les N-Reines ressemble à ceci :

// Variables : une par colonne
IntVar[] q = makeIntVarArray(cp, n, n); // n variables, domaine {0..n-1}

// Contraintes : pas sur la même rangée
cp.post(new AllDifferent(q));

// Contraintes : pas sur la même diagonale
for (int i = 0; i < n; i++)
  for (int j = i+1; j < n; j++) {
    cp.post(new NotEqual(q[i], q[j], i-j)); // diag
    cp.post(new NotEqual(q[i], q[j], j-i)); // anti-diag
  }
