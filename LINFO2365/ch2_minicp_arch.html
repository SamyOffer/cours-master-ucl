📦 Les Domaines — Qu'est-ce qu'un domaine ?

🧒 C'est quoi un domaine ?

Un domaine, c'est l'ensemble des valeurs qu'une variable peut encore prendre. Au début, si ta variable représente "le chiffre d'une case de Sudoku", son domaine est {1, 2, 3, 4, 5, 6, 7, 8, 9}. Au fur et à mesure que tu résous le problème, des valeurs sont éliminées du domaine.

Règle d'or : On ne fait qu'enlever des valeurs d'un domaine, jamais en ajouter. Si le domaine devient vide → contradiction → backtrack !

💡 Le domaine = ce qui reste possible. Plus il rétrécit, plus on est proche d'une solution (ou d'un échec).
🔍

contains(v)

v est-il encore dans le domaine ?

➖

remove(v)

Supprimer v du domaine

📏

size()

Nombre de valeurs restantes

✅

isFixed()

Le domaine est-il un singleton ?

⚡ Le Sparse Set — Structure de données O(1)

🧒 C'est quoi un Sparse Set ?

Imagine une liste de personnes dans une salle. Si quelqu'un sort, au lieu de décaler tous les autres (lent !), tu mets la dernière personne à la place de celle qui est sortie. Ça prend O(1) — un seul échange !

C'est exactement ça, le Sparse Set. C'est un tableau avec deux pointeurs (values et indices) qui permet de retirer des éléments en temps constant, ET de revenir en arrière (backtrack) facilement en restaurant juste un entier.

Deux tableaux : values[i] = la valeur à la position i. indices[v] = la position de la valeur v. Et size = le nombre d'éléments actifs (tout ce qui est avant l'indice size est "dans" le domaine).

💡 Propriété clé : ∀v ∈ {0..n-1} : values[indices[v]] = v
📐 Invariant du Sparse Set

📌 "Les valeurs actives sont toujours dans les positions [0, size-1] du tableau values"

TermeSignification
values[i]La valeur à la position i dans le tableau
indices[v]La position de la valeur v dans le tableau
sizeNombre d'éléments actifs. Les éléments actifs sont values[0..size-1]
remove(v)Échanger v avec values[size-1], puis décrémenter size → O(1)

Pourquoi pas juste un HashSet ? Un HashSet standard ne supporte pas le backtrack efficacement — pour défaire les suppressions, il faudrait logger chaque opération séparément. Le Sparse Set avec StateInt size (StateInt = entier dont la valeur est sauvegardée/restaurée automatiquement) permet le backtrack en O(1) : on restaure juste la valeur de size !

🔗 Variables & Propagation — Comment les contraintes communiquent

🧒 C'est quoi la propagation ?

Imagine un Sudoku : quand tu places un "5" dans une case, tu dois immédiatement retirer le "5" de toutes les cases de la même ligne, colonne et région. C'est ça, la propagation : un changement dans un domaine déclenche automatiquement des vérifications dans les domaines liés.

Dans MiniCP, chaque contrainte s'abonne aux événements des variables (ex: "préviens-moi quand x est fixé"). Quand un événement se produit, la contrainte se réveille et réexécute son filtre.

💡 Propagation = réaction en chaîne d'éliminations. Comme des dominos qui tombent.

  • post() : la contrainte s'enregistre pour recevoir des notifications. Ex : "quand x change, appelle propagate()"

  • propagate() : la contrainte examine ses variables et retire les valeurs qui violent la contrainte

  • Si un domaine change → les autres contraintes liées sont réveillées → elles propagent à leur tour

  • Le tout se répète jusqu'au fixpoint : plus aucune contrainte ne peut éliminer de valeur

Événements supportés dans MiniCP :

• propagateOnDomainChange(c) : appeler c.propagate() quand le domaine change

• propagateOnFix(c) : appeler c.propagate() quand la variable est fixée

• propagateOnBoundChange(c) : appeler c.propagate() quand min ou max change

🔄 L'Algorithme de Fixpoint

🧒 C'est quoi le fixpoint ?

Pense à une machine à laver : elle continue de tourner jusqu'à ce que le linge soit propre (= plus rien à faire). Le fixpoint, c'est pareil : le solveur continue de propager les contraintes jusqu'à ce qu'aucune nouvelle valeur ne puisse être éliminée.

File de propagation : Quand une contrainte modifie un domaine, elle ajoute les contraintes affectées dans une file d'attente. Le solveur vide cette file, une contrainte à la fois.

💡 Le fixpoint garantit qu'on a tiré tout ce qu'on pouvait des contraintes avant de faire le prochain choix dans l'arbre de recherche.

  • Ajouter toutes les contraintes actives dans une file de propagation

  • Prendre la première contrainte de la file, appeler propagate()

  • Si propagate() modifie un domaine → les contraintes liées sont ajoutées à la file

  • Répéter jusqu'à ce que la file soit vide → fixpoint atteint

  • Si un domaine devient vide → lever une exception InconsistencyException → backtrack

💾 StateManager & Backtrack — Revenir en arrière

🧒 C'est quoi le StateManager ?

Imagine que tu joues à un jeu vidéo et que tu peux sauvegarder à tout moment. Si tu meurs, tu charges ta sauvegarde et tu recommences depuis ce point. Le StateManager, c'est exactement ça pour MiniCP.

Chaque fois qu'on explore une branche de l'arbre de recherche, on sauvegarde l'état (les tailles des domaines). Si on arrive à un échec, on restaure l'état (backtrack) et on essaie une autre valeur.

Les StateInt sont des entiers "magiques" qui savent comment se sauvegarder et se restaurer automatiquement.

💡 StateManager = système de sauvegarde/restauration pour les recherches DFS avec backtrack.
